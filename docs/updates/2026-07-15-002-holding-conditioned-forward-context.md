# 持仓基金同状态历史表现

## 1. 更新信息

- 更新编号：`2026-07-15-002`
- 策略：`fund_conditioned_forward_return@1.0.0`
- 接口：`GET /api/holdings/level-recurrence`
- 数据：东方财富基金净值走势 / 天天基金历史净值
- 展示位置：我的资产 -> 持仓与纪律 -> 单只基金详情 -> 真实市场证据

## 2. 目标

“当前估值上一次何时到达”只回答了时间和价位，不能说明同类市场状态之后的历史结果。本次更新把已有的版本化同状态前瞻策略接入逐项持仓详情，用真实确认净值回答：

1. 当前确认净值位于 MA60 上方还是下方。
2. 当前处于接近高位、普通回撤还是深度回撤区。
3. 历史上具有相同趋势和回撤区间的月末样本，在随后 3、6、12 个月分别如何分布。
4. 同状态分布相对基金自身全部历史月末样本是否存在差异。

该结果是研究证据，不是未来收益预测，也不直接生成仓位动作。

## 3. 数据流

```text
用户已确认基金持仓
  -> 东方财富盘中估值
  -> 最长 120 个月真实确认净值（单次读取）
  -> 当前估值历史到达/穿越
  -> 最新确认净值状态分类
  -> 历史同状态月末样本
  -> 3/6/12 月真实前瞻收益分布
  -> 批量持仓响应
  -> 单只持仓详情
```

估值穿越和同状态前瞻共用同一批确认净值，避免为新增视图再次请求上游数据。单只基金失败只影响该行，其他基金和股票继续返回。

## 4. 计算口径

### 4.1 当前状态

- 趋势：最新确认净值相对最近 60 个确认净值观察值的均值。
- 回撤区间：相对当时历史高点的回撤。
- 回撤分层：接近阶段高位、普通回撤、深度回撤。
- 盘中估值不参与状态分类，避免把未确认数据写入历史策略。

### 4.2 历史样本

- 每个自然月只取最后一个确认净值观察值。
- 历史样本必须同时匹配当前趋势和回撤区间。
- 每个样本只使用该日期之前可见的数据进行状态分类，防止未来数据泄漏。
- 3、6、12 个月分别使用 63、126、252 个确认净值观察值作为前瞻窗口。

### 4.3 输出指标

- 正收益占比。
- 平均收益和中位收益。
- 25% 与 75% 分位数，即中间 50% 区间。
- 最好和最差历史结果。
- 同状态样本相对全部历史月末样本的正收益占比差和中位收益差。
- 样本数量、开始日期和结束日期。

## 5. 门禁

1. 同状态样本少于 6 个时，窗口状态为 `insufficient_samples`。
2. 所有窗口都不足时，策略状态为 `insufficient_data`，不得展示方向判断。
3. 确认净值源失败或计算异常时返回 `unavailable`，不得生成模拟样本或模板结论。
4. 当前策略注册状态仍为 `shadow`，只允许计算和展示研究证据，不得驱动金额。
5. 股票不复用基金净值策略，不显示伪造的同状态分布。

## 6. 前端表达

持仓列表保持一行一项，不再增加列。用户点击基金后才显示：

- 最新确认净值日期、净值、MA60 状态和回撤区间。
- 主要观察窗口和数据置信等级。
- 3/6/12 个月的正收益占比、中位收益、中间 50% 区间及样本数。
- “影子研究，不驱动仓位”和“历史分布不是未来预测”的固定门禁文案。

“同状态样本偏正”只能表示历史分布，不等于加仓；“同状态样本偏弱”只触发风险复核语义，不等于自动卖出。

## 7. 已知限制

- 3/6/12 个月前瞻窗口可能重叠，样本并非完全独立。
- 基金经理、投资合同、跟踪指数和费率变化尚未标准化。
- 只与基金自身无条件历史比较，不是可投资同类基准。
- QDII 确认净值可能滞后，其状态日期必须按页面显示的确认净值日期理解。
- 历史分布不能证明未来仍处于相同市场机制。

## 8. 验收条件

- 基金估值与同状态分布共用一次真实确认净值读取。
- 批量接口保留基金策略版本、状态和完整窗口结构。
- 样本不足和真实来源失败均明确拒绝方向结论。
- 股票结果不携带基金策略对象。
- 持仓详情能在桌面和窄屏布局中完整展示，且无横向溢出。
- 后端全量测试、前端生产构建和真实基金请求全部通过。
