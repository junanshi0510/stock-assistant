# 基金替代候选真实费率与披露持仓门禁

## 1. 目标

上一阶段已经解决“不能只看近一年榜单换基”的问题，但持续胜率通过仍不能证明值得替换。候选可能：

- 与当前基金持有同一批股票，只是名称不同；
- 处于相同宽泛行业，但实际股票并不相同；
- 年度运作费更高；
- 页面申购费较低，但当前基金赎回费尚未满足免收期限；
- 缺少最新可得定期报告，无法判断底层暴露；
- 基金经理或合同已经变化，历史持续性不再对应当前载体。

本次新增第二阶段门禁，只回答“是否值得继续核对用户实际换仓成本”，不回答“是否应当卖出或买入”。

## 2. 诊断版本

- 诊断 ID：`fund_alternative_due_diligence`
- 版本：`1.0.0`
- 上游依赖：`fund_alternative_durability@1.0.0`
- 自动买入：固定禁止
- 自动赎回：固定禁止
- 自动切换：固定禁止

## 3. 真实数据

### 3.1 费率

来源为天天基金基金费率档案页，保留：

- 管理费率；
- 托管费率；
- 销售服务费率；
- 申购金额分档、原费率和页面优惠费率；
- 固定每笔费用；
- 赎回持有期限分档和对应费率。

解析使用 HTML 表格结构，不按页面整段文本猜测。运作费、申购费或赎回费任一关键表缺失时，该候选标记为费率证据不完整。

年度明确运作费率为管理费、托管费和销售服务费之和。基金页面已说明这些费用从基金资产中计提、净值已经扣除，因此系统不会再从历史净值收益中重复扣减。

页面优惠申购费只代表该页面对应销售渠道和金额分档，不推断其他平台执行相同折扣。

### 3.2 披露持仓

来源为基金最新可得定期报告；ETF 联接继续沿用已经验证的目标 ETF 一层穿透。

股票重合下界：

```text
Σ min(当前基金披露股票占净值比例, 候选基金披露股票占净值比例)
```

行业重合下界使用同一公式。结果只覆盖双方已披露部分，不能解释为完整实时组合重合度。

### 3.3 基金经理

基金经理姓名、任职信息、页面能力评分和任期相对同类结果来自基金详情页，只作为证据完整性检查，不直接形成买卖分数。

## 4. 重合度规则

版本 `1.0.0` 使用以下阈值：

| 维度 | 中度 | 高度 |
|---|---:|---:|
| 共同披露股票 | 8% | 20% |
| 共同披露行业 | 45% | 70% |

当双方都有股票披露时：

- 共同持股达到 20%，判定高度重合；
- 共同持股达到 8% 且行业达到 70%，判定高度重合；
- 只有宽泛行业达到 70%，但共同持股低于 8%，只判定中度风格重合。

当股票披露不可用、只能比较行业时，行业达到 70% 才判定高度重合。

该规则避免把“共同持股为零，但都被归入制造业”的基金误判为同一底层组合。

## 5. 费率规则

- 候选年度明确运作费率比当前基金低至少 `0.10` 个百分点，记为明确成本优势；
- 候选年度明确运作费率高至少 `0.30` 个百分点，记为明显成本溢价；
- 历史净值收益已扣除运作费，不将年度费率再次从历史超额中扣减；
- 没有用户逐笔买入日期和剩余份额时，实际赎回费保持 `null`；
- 不使用最低赎回费、最长持有期费率或经验值替代用户真实费率。

## 6. 状态机

| 状态 | 含义 | 允许继续用户成本核验 |
|---|---|---|
| `blocked_by_durability` | 持续性门禁未通过 | 否 |
| `insufficient_disclosure` | 定期报告持仓不足 | 否 |
| `incomplete_fee_evidence` | 关键费率表不完整 | 否 |
| `duplicate_without_cost_edge` | 高度重合且没有年度费率优势 | 否 |
| `duplicate_but_cost_edge` | 高度重合，但候选年度费率明确更低 | 是 |
| `distinct_but_costlier` | 底层暴露有差异，但年度费率明显更高 | 是，需证明能力覆盖成本 |
| `partial_overlap_candidate` | 部分风格重合 | 是 |
| `distinct_candidate` | 披露持仓重合下界较低且费率未显著恶化 | 是 |

“允许继续”只代表可以读取用户交易批次和平台报价，绝不代表换仓建议。

## 7. 用户界面

原替代候选区域增加第二条汇总带：

- 已验证数量；
- 进入持有期成本核验数量；
- 高重合且无成本优势数量；
- 费率缺口数量。

每只候选行内展示：

- 持续性状态；
- 替换价值状态；
- 6/12 个月滚动胜率；
- 12 个月中位超额；
- 股票重合下界；
- 年度明确费率差；
- 候选年度明确运作费率；
- 主要共同披露持股；
- 下一步尚缺的用户级成本证据。

Agent 工作台使用同一个 Evidence，不重新计算另一套重合或费率结论。

## 8. Agent 约束

模型上下文只接收裁剪后的：

- 诊断状态和解释；
- 重合下界摘要及少量共同持股；
- 年度费率差；
- 经理摘要；
- 门禁结果和剩余缺口。

完整赎回费表、原始持仓列表和页面 HTML 不发送给模型。

系统提示词要求：

- 持续性未通过时不得称为可换入机会；
- 第二阶段未通过时不得称为替换候选；
- 第二阶段通过也只能称为进入持有期成本核验；
- 实际赎回费为空时必须声明用户逐笔持有天数和平台报价未知；
- 不得生成自动换仓、买入或赎回指令。

## 9. 失败语义

- 费率页面不可用：保留 `real_fee_schedule_unavailable`；
- 定期报告不可用：保留 `real_periodic_portfolio_unavailable`；
- 单只候选失败不影响其他候选；
- 缺失持仓不能解释为零重合；
- 缺失费率不能解释为零费用；
- 顶层返回 `partial`，不使用模拟或同类平均补齐。

## 10. 后续阶段

下一阶段应绑定用户真实交易流水：

1. 按 FIFO 识别剩余份额的实际持有天数；
2. 匹配费率档案中的赎回期限分档；
3. 接受用户确认的销售平台当日费率报价；
4. 计算赎回费、候选申购费、到账等待和市场在途风险；
5. 只在用户投资政策、仓位上限和现金流约束同时通过时形成分批执行草案。

在这些证据完成前，系统不会把候选转换为交易指令。
