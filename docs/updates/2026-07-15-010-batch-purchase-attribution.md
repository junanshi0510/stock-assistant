# 2026-07-15 更新 010：批量基金真实成交后绩效归因

## 1. 目标

批量基金决策链已经能够完成组合分配、执行前复核、真实成交绑定和 FIFO 持仓对账。本次继续回答成交后的核心问题：本批次实际投入后来产生了多少已实现收益、剩余份额价值和总收益，以及当前证据是否足以进入策略复盘。

本功能不分析未成交基金的假设收益，不使用盘中估值，不用当前整只基金持仓收益冒充本批次收益，也不把单次正收益解释为策略长期有效。

## 2. 主链位置

```text
单基金 Evidence Run
  -> 批次披露持仓重合
  -> 组合资金分配
  -> 销售平台执行前复核
  -> 真实成交流水绑定
  -> 当前持仓与 FIFO 份额对账
  -> 本批次真实成交绩效快照
  -> 满 30 天后进入策略有效性复盘
```

绩效快照是独立的追加式审计链。刷新净值不会改写成交事件或之前的绩效快照。

## 3. 真实收益口径

每只基金只追踪执行事件中绑定的具体买入流水：

```text
原始成本 = 买入确认份额 × 买入确认净值 + 买入费用
已实现回款 = 后续卖出按 FIFO 分配给该买入批次的净回款
剩余份额价值 = 该买入批次 FIFO 剩余份额 × 最新确认单位净值
批次结束价值 = 已实现回款 + 剩余份额价值
批次总收益 = 批次结束价值 - 原始成本
```

卖出费用按卖出流水的实际匹配比例扣除。全部卖出且当前持仓记录已经删除时，只要 FIFO 账本完整，仍可计算最终已实现收益。

## 4. 数据门禁

生成快照前必须同时满足：

1. 批次真实成交事件和持仓对账事件的哈希链完整。
2. 绑定买入流水仍存在、属于当前用户且内容哈希未变化。
3. 该买入流水的原始份额等于 FIFO 已卖出份额与剩余份额之和。
4. 当前确认持仓总份额与该基金全部 FIFO 未平仓份额一致；已清仓时双方均为零。
5. 买入确认净值与公开数据源同日确认单位净值的偏差不超过 `max(0.0001, 0.2%)`。
6. 后续卖出确认净值也必须通过同日净值交叉验证。
7. 仍有剩余份额时，最新确认净值不能超过 10 天未更新。
8. 持有区间存在未进入交易账本的分红或拆分时停止收益归因。

任一基金证据不完整时，不为该基金生成收益数字。批次存在部分不可用基金时，不把已覆盖部分包装成整批收益。

## 5. 时间和策略评价

真实资金结果与策略有效性评价分开：

- 净值和账本完整时，可以立即展示截至真实确认净值日的资金结果。
- 买入后不足 30 个自然日时只标记为成交后监控。
- 全部真实成交均满 30 天且证据完整后，才开放策略复盘资格。
- 单批次结果只用于复盘原决策，不能证明未来收益或策略具备持续超额能力。

## 6. 不可变快照链

新增：

- `agent_batch_purchase_attribution_snapshots`
- `portfolio_batch_purchase_attribution@1.0.0`
- `POST /api/v1/agent/batches/{batch_id}/purchase-attribution`

每个快照绑定：

- Batch 输入哈希；
- 原真实成交事件 ID、事件哈希和 Payload 哈希；
- 原持仓对账事件 ID、事件哈希和 Payload 哈希；
- 相关基金完整交易账本哈希；
- 当前确认持仓份额哈希；
- 真实净值和分红拆分数据哈希；
- 前一绩效快照哈希。

数据库触发器禁止更新或删除绩效快照。相同请求保持幂等，并通过前序哈希阻止并发覆盖。

## 7. 页面

批量基金结果页在“真实成交与持仓对账”之后新增“成交后绩效归因”：

- 批次实际成本、已实现回款、剩余份额价值、实际收益和收益率；
- 每笔批次买入的已卖出/剩余份额、已实现/未实现收益；
- 真实净值截止日、观察天数、最大回撤；
- 净值来源链接、不可用原因、账本当前性和快照审计状态；
- 30 天策略复盘门槛。

历史批次列表仍不扫描交易账本或加载绩效明细，只有进入单个批次详情时才计算本地完整性状态。

## 8. 明确不做

- 不推断未成交基金如果买入会赚多少。
- 不使用当前整仓金额按比例猜测批次收益。
- 不补齐缺失分红、拆分、费用、份额或净值。
- 不因历史正收益生成加仓指令。
- 不连接销售账户或自动交易。
