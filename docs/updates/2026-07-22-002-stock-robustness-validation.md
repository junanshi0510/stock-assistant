# 2026-07-22 更新 002：股票策略历史稳健性体检

## 1. 解决的问题

成本后执行回测已经能回答“一组参数在一段历史里是否形成正净期望”，但单次正结果仍可能来自参数选择偏差、某一段行情、偏低的成交成本或数据末端截断。若继续把历史上最好看的止损、止盈和入场阈值作为推荐，回测会比真实决策乐观。

本轮新增 `stock_signal_robustness@1.0.0`。它不预测必涨股票，也不优化出“最高收益参数”，而是固定围绕用户当前参数做多维否决检查，优先回答：

- 邻近参数稍微变化后，正净期望是否消失；
- 同一组参数在较晚历史时间里是否仍成立；
- 提高佣金、滑点和卖出税费后是否仍有正净期望；
- 结果是否只依赖某一小段行情；
- 是否有足够非重叠交易支持上述比较。

## 2. 固定原则

```text
用户当前参数
  -> 完整历史基线成本后门槛
  -> 参数邻域覆盖与通过率
  -> 较早 60% / 较晚 40% 时间留出
  -> 更高成交成本场景
  -> 四段时间一致性
  -> 按固定否决顺序给出稳健性状态
```

系统不会按策略收益、净期望或回撤从矩阵中选出“最佳组合”，前端也不突出最强场景，只展示最弱值、中位数、最强值和全部原始组合。当前参数始终单独标记。

## 3. 完整持有期边界

所有执行比较现在只接收能够在所属时间窗口内完整走完最长持有期的信号。对于左闭右开的窗口 `[start, end)` 和最长持有期 `horizon`：

```text
允许的最晚信号索引 = end - horizon - 1
```

因此：

- 基线不会在数据末尾生成被提前截断的到期退出；
- 开发段交易不会进入较晚留出段；
- 四个时间段之间不会共享或跨界退出；
- 交易仍固定在信号后的下一交易日开盘入场。

这项修正避免数据尾部短持仓被错误当作完整策略结果。

## 4. 参数邻域

默认中心点为用户提交的 `entry_score`、`stop_atr` 和 `target_atr`，分别检查：

| 轴 | 邻域 |
| --- | --- |
| 入场分数 | 当前值 `-5 / 0 / +5` |
| 止损 ATR | 当前值 `-0.5 / 0 / +0.5` |
| 止盈 ATR | 当前值 `-0.5 / 0 / +0.5` |

在参数未触及 API 上下界时一共形成 27 组。触及边界后先裁剪并去重，不虚构超出公开接口范围的参数。

每组至少需要 20 笔非重叠交易才可评估。稳健性总门槛还要求至少 60% 的邻域组合可评估，并要求可评估组合中至少 70% 同时满足：

- 单笔成本后净期望大于 0；
- 盈利因子大于 1；若达到样本门槛且历史交易没有亏损笔，盈利因子无法形成分母时不单独否决。

未达到 20 笔的组合保留在矩阵中并标记“样本不足”，不会计入正面通过数。

## 5. 时间检查

### 60/40 时间留出

可评分历史按时间顺序切分：较早 60% 为开发段，较晚 40% 为时间留出段。两段使用完全相同的用户参数，不在开发段重新优化。较晚段至少需要 10 笔完整、非重叠交易，并必须保持正净期望和通过盈利因子条件。

该结果被明确称为“时间留出”而不是绝对样本外：用户一旦看过较晚段结果并据此继续调参，该后段就已经参与参数选择，不能继续被当作未见数据。

### 四段一致性

同一历史区间再按时间等距切为四段。每段至少 5 笔交易才可评估；至少 3/4 段必须可评估，可评估段中至少 50% 必须保持历史正净期望。该检查用于发现结果是否集中在单一行情阶段，不等同于市场状态分类或滚动训练。

## 6. 成本压力

高成本场景保持用户的入场、止损、止盈、风险预算和仓位上限不变，只提高成本输入：

```text
压力值 = min(API 上限, max(当前值 × 2, 当前值 + 5 bps))
```

佣金、滑点和卖出税费分别计算。该场景至少需要 20 笔交易，并必须保持成本后正净期望和盈利因子条件。它只是统一压力值，不代替用户券商、市场和账户的真实费用确认。

## 7. 固定否决顺序

总门槛不做加权平均，按以下顺序返回第一个主要失败原因：

1. `baseline_not_positive`：完整历史基线尚未通过；
2. `insufficient_evidence`：参数覆盖、较晚段、高成本或分段样本不足；
3. `chronological_holdout_failed`：较晚 40% 未保持正净期望；
4. `cost_stress_failed`：提高成本后未通过；
5. `parameter_fragile`：邻域参数正面通过率低于 70%；
6. `time_inconsistent`：分阶段正面通过率低于 50%；
7. `historically_robust`：以上历史门槛全部通过。

任何失败状态都不会被另一个高收益场景抵消。`historically_robust` 也只是一项历史研究标签，不是买入信号、统计显著性证明或收益承诺。

## 8. API 与前端

`GET /api/backtest` 路径和原参数保持兼容，在原 `execution` 之外新增 `robustness`：

- `policy_version` 与总 `gate`；
- 固定 `criteria`；
- 参数轴、汇总分布和全部场景；
- 60/40 开发段与时间留出段；
- 四个独立时间段；
- 高成本假设和结果；
- 数据污染、偏差与实盘边界警告。

“研究中心 → 历史策略验证”新增：

- 稳健性总门槛和精确失败原因；
- 参数、后段时间、高成本和时间一致性四张摘要卡；
- 60/40 时间留出与四段明细表；
- 参数净期望最弱值、中位数和最强值；
- 默认折叠的全部参数矩阵；
- 独立的限制说明。

宽表只在自身容器内横向滚动，不扩大手机页面宽度。

## 9. 验证范围

- 专项测试由 5 项增加到 9 项，新增覆盖完整 27 组参数矩阵、时间留出、四段一致性、高成本输入、完整持有期边界和后段失败不可被其他结果掩盖。
- 后端全量回归 `430 tests` 通过，前端 Vite 生产构建通过。
- 真实 A 股 `600519` 形成 907 个方向样本和 42 笔完整持有期交易，基线为 `non_positive_expectancy`，稳健性总门槛返回 `baseline_not_positive`；27 组参数均达到样本门槛，但没有一组保持历史正期望。
- 真实港股 `00700` 形成 918 个方向样本和 44 笔交易，基线同样为 `non_positive_expectancy`；27 组参数均可评估，其中 25.9% 为历史正期望，总门槛仍固定返回 `baseline_not_positive`。
- 真实美股 `AAPL` 形成 940 个方向样本和 44 笔交易。完整历史基线为 `historically_positive`，27 组邻域也全部为历史正期望，但较晚 40% 只有 15 笔交易、净期望 `-0.030%`、盈利因子 `0.988`，最终被 `chronological_holdout_failed` 否决。这一结果验证了稳健性门槛不会让全样本正收益覆盖时间失效证据。
- Chrome 桌面端确认 27 组参数全部可展开且只有 1 组标记当前参数；`390×844` 手机端页面整体无横向溢出，4 个宽表均限制在局部滚动容器，浏览器控制台无错误或警告。

## 10. 仍未解决的边界

- 没有自动执行组合式对称交叉验证、Deflated Sharpe Ratio 或多重检验校正；
- 没有股票池历史成分，仍可能存在幸存者偏差；
- 没有按牛熊、波动率、流动性和宏观状态训练独立参数；
- 时间留出可以被用户反复调参污染，不是永久有效的真正样本外；
- 没有前瞻纸面交易、实盘成交回报或券商对账；
- 仍未覆盖停牌、涨跌停排队、整手、公司行为、汇率、融资成本和市场冲击；
- 单股票历史稳健不代表组合层面的相关性、容量或尾部风险已经通过。

## 11. 设计依据

- Bailey、Borwein、López de Prado 与 Zhu 的原始论文说明，投资回测在反复选择策略时容易过拟合，简单留出法本身也可能不可靠，因此本轮把时间留出和参数邻域作为最低压力检查，同时明确它们不是完整 PBO 校正：[The Probability of Backtest Overfitting](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=2326253)。
- CFA Institute 的回测与仿真课程说明滚动窗口/前推框架可用于按时间检验策略，并提示前视偏差、幸存者偏差、肥尾与结构突变风险；本轮据此加入分段与场景检查，但不声称已覆盖全部偏差：[Backtesting & Simulation](https://www.cfainstitute.org/insights/professional-learning/refresher-readings/2026/backtesting-and-simulation)。
- 美国 SEC 投资顾问营销规则要求假设性表现提供足够的假设、风险和限制背景，并禁止不公平或误导性的业绩呈现；因此页面同时展示净结果、方法版本、样本门槛和失败原因：[Investment Adviser Marketing](https://www.sec.gov/resources-small-businesses/small-business-compliance-guides/investment-adviser-marketing)。
