# 2026-07-22 更新 001：股票信号成本后执行回测

## 1. 解决的问题

原历史验证只回答“某天的看涨/看跌分数与之后 N 日涨跌是否同向”。这些样本每天滚动、彼此重叠，而且默认按信号日收盘价观察未来收益，不能直接解释为真实可成交策略，更没有回答以下问题：

- 信号只能在收盘后确认，下一交易日开盘再交易会怎样；
- 佣金、滑点和卖出税费是否会吃掉表面收益；
- 止损、止盈和最长持有期会产生多少真实交易；
- 按账户风险预算倒推仓位后，复利结果和最大回撤是多少；
- 日线同时触及止损与止盈、或跳空穿透止损时如何保守处理。

本轮保留原有方向统计作为“信号诊断”，新增独立的 `stock_signal_execution_backtest@1.0.0` 执行仿真。两个口径同时返回，但前端明确分区，不把重叠方向样本冒充交易收益。

## 2. 固定执行顺序

```text
历史日线截至信号日
  -> 使用当日及更早数据计算技术分
  -> 分数达到用户阈值
  -> 下一交易日开盘价进入多头仿真
  -> 使用信号日 ATR 固定止损和止盈
  -> 逐日检查跳空、止损、止盈和最长持有期
  -> 扣除买卖双边佣金/滑点与卖出税费场景值
  -> 按计划止损损失倒推账户仓位
  -> 退出后才允许寻找下一笔信号
  -> 汇总净期望、盈利因子、复利净值与回撤
```

信号日不会按当日收盘成交，持仓期间不会叠加第二笔同标的仓位。执行仿真当前只做多；看跌分数仍只参与原方向统计，不会被解释为可融券或可卖空许可。

## 3. 用户可调参数

`GET /api/backtest` 保持原路径，新增以下查询参数：

| 参数 | 范围 | 默认场景 | 含义 |
| --- | ---: | ---: | --- |
| `entry_score` | 50-90 | 65 | 允许进入多头仿真的最低技术分 |
| `horizon` | 3-60 | 20 | 单笔最长持有交易日 |
| `stop_atr` | 0.5-6 | 2 | 信号日 ATR 止损倍数 |
| `target_atr` | 0.5-12 | 3 | 信号日 ATR 止盈倍数 |
| `commission_bps` | 0-100 | 5 | 每一边佣金场景值 |
| `slippage_bps` | 0-100 | 5 | 每一边滑点场景值 |
| `sell_tax_bps` | 0-200 | 0 | 仅卖出侧税费场景值 |
| `risk_per_trade_pct` | 0.1%-5% | 1% | 每笔计划账户风险预算 |
| `max_position_pct` | 1%-100% | 30% | 单股仿真仓位上限 |

基点输入全部是用户场景假设。默认值不是任何市场、券商或账户的真实收费规则；用户必须按自己的交易通道调整。

## 4. 成交与路径规则

### 下一日开盘

技术分使用信号日收盘后才能完整获得，因此固定使用下一可用交易日开盘价作为原始入场价，再叠加买入侧佣金和滑点。这样避免把尚未形成的收盘信号用于同一收盘价成交。

### 止损与止盈

入场后固定：

```text
止损价 = 入场开盘价 - stop_atr × 信号日 ATR
止盈价 = 入场开盘价 + target_atr × 信号日 ATR
```

后续交易日若开盘已经穿透止损或止盈，使用实际开盘价退出，不假设仍能按原触发价成交。

日线 OHLC 无法证明同一天内高点和低点的先后顺序。当一根 K 线同时触及止损与止盈时，固定按止损先发生处理，并在结果中累计 `same_bar_ambiguous_count`，不选择对回测更有利的路径。

### 仓位倒推

系统先计算包含成本的计划止损百分比，再按以下关系倒推仓位：

```text
计划仓位% = min(单股仓位上限%, 单笔账户风险预算% / 计划止损损失% × 100)
```

跳空可能让实际退出价低于止损价，因此真实仿真损失仍可能超过风险预算。此类交易会标记 `risk_budget_breached`，系统不会把止损描述为亏损保证。

## 5. 返回指标

`execution` 固定返回：

- 毛收益和成本后净收益的单笔期望；
- 胜率、盈利因子、平均盈亏比和平均成本拖累；
- 按风险仓位逐笔复利的策略结果和最大回撤；
- 止盈、止损、到期退出数量；
- 平均持有天数、平均仓位和风险预算穿透次数；
- 最多最近 30 笔完整路径，包括信号/入场/退出日期、价格、MFE、MAE、仓位和账户贡献；
- 全部交易退出时点的净值与回撤序列；
- 固定方法版本、执行假设和未覆盖风险。

研究门槛不生成买卖指令：

- 少于 20 笔非重叠交易：`insufficient_samples`；
- 净期望不为正或盈利因子不高于 1：`non_positive_expectancy`；
- 同时满足最低样本、净期望为正和盈利因子高于 1：`historically_positive`。

`historically_positive` 只表示当前单组参数在当前历史样本中的成本后结果为正，不表示统计显著、样本外有效或未来会盈利。

## 6. 前端交互

“研究中心 → 历史策略验证”现在提供：

1. 可编辑的入场分数、ATR 止损止盈、佣金、滑点、卖出税费、单笔风险和仓位上限；
2. 成本后研究门槛与精确策略版本；
3. 复利交易净值曲线、最大回撤、净期望和盈利因子；
4. 退出原因分布、仓位与持有期；
5. 最近交易路径和跳空/同日双触发标记；
6. 与执行结果严格分开的原方向准确率、分数分档和未来 N 日统计。

移动端参数改为双列，宽交易表使用局部横向滚动，不扩大整个页面宽度。

## 7. 验证

- 新增 5 个确定性单元测试，覆盖下一日开盘、防前视成本扣减、同日止损止盈保守顺序、跳空穿透风险预算、完整方向统计兼容和参数边界。
- 路由契约与 market-data Worker 网关回归通过，公开 API 路径没有变化。
- 后端全量回归 `426 tests` 通过，前端 Vite 生产构建通过。
- 真实 A 股 `600519` 冒烟验证读取约四年日线，形成 907 个方向样本和 43 笔非重叠交易；默认成本/风控场景返回 `non_positive_expectancy`，证明系统会拒绝把负历史期望包装成机会。
- 港股 `00700` 与美股 `AAPL` 同样完成约四年真实日线执行冒烟，两个市场都生成非重叠交易和研究门槛结果；验证既包含负期望结果，也包含正期望结果，不统一输出乐观标签。
- Chrome 桌面端和 `390×844` 手机端完成表单、研究门槛、结果卡、净值曲线、交易路径和局部横向滚动检查；页面无整体横向溢出，控制台无错误或警告。

## 8. 明确不做

- 不连接券商，不读取余额，不提交订单。
- 不模拟停牌、涨跌停排队、整手限制、分红、拆股、配股、汇率、融资利息和市场冲击。
- 不从日线推断真实盘中先后路径；歧义路径固定保守处理。
- 不把单次参数结果当作参数优化、滚动样本外验证或概率校准。
- 不把回测收益解释为用户真实盈亏或未来收益。

## 9. 设计依据

- 美国 SEC 投资者教育材料说明交易费用和持续费用都会降低投资回报，因此本轮把成本作为显式输入并同时展示毛/净结果：[How Fees and Expenses Affect Your Investment Portfolio](https://www.investor.gov/introduction-investing/general-resources/news-alerts/alerts-bulletins/updated)。
- SEC 对假设性、回测表现的监管案例强调：若未披露回测与真实执行的差异，结果可能误导，因此页面固定展示成交假设和未覆盖限制：[SEC Charges Investment Adviser with Compliance and Books and Records Failures](https://www.sec.gov/enforcement-litigation/administrative-proceedings/ia-5945-s)。
- FINRA 的收益计算说明要求把投资费用计入总成本，本轮的净收益与账户贡献均先扣除用户设置的成本场景：[Calculating Your Investment Returns](https://www.finra.org/investors/insights/investment-returns)。
