# 2026-07-23 策略收益实验室与资金资格引擎

## 1. 为什么这是一个大功能

机会工厂此前已经能完成候选发现、真实数据门禁、五因子横向比较、约束后组合和纸面冻结，但它仍缺少决定“是否值得投入研究资金”的关键后半段：

1. 纸面组合需要用户手工刷新，无法持续积累前瞻证据。
2. 观察结果只有组合收益，没有同市场基准、成本和净超额。
3. 第 5/20/60 日没有固定取样；服务中断后补跑可能把更晚日期误当目标窗口。
4. 同一天重复运行可被误算为多个“独立样本”。
5. 同时尝试大量策略再挑赢家，会产生多重检验和选择偏差。
6. 策略分数、回测或单次上涨与个人资金预算之间没有可信门禁。

本版本新增“策略收益实验室与资金资格引擎”，把主链补全为：

```text
版本化机会策略
  -> 不可变扫描结果
  -> 冻结纸面组合
  -> 自动真实行情观察
  -> 精确第 5/20/60 个交易日
  -> 往返成本后净收益
  -> 同市场基准净超额
  -> 独立批次与跨策略统计校正
  -> 资金资格门禁
  -> 受 IPS/预算/可信估值约束的人工试运行上限
```

它不是“预测明天一定涨”的模型。它解决的是更可验证的问题：一个固定策略在真正冻结之后，能否在成本后持续胜过相应市场，以及证据是否足以允许极小比例的人工试运行。

## 2. 同行产品调查后的取舍

本轮参考了以下官方产品能力：

- [TradingView Portfolios](https://www.tradingview.com/support/solutions/43000760937-tradingview-portfolios-track-your-assets-know-your-trades/) 强调组合跟踪、交易记录和实际表现，而不只停留在筛选结果。
- [Koyfin Model Portfolios](https://www.koyfin.com/help/model-portfolios/) 把模型组合、基准和组合分析放在同一研究流程。
- [Sharesight Portfolio Performance](https://help.sharesight.com/show_portfolio/) 强调真实组合收益、资本利得、分红和基准比较。
- [Snowball Analytics Overview](https://snowball-analytics.com/overview) 把目标配置、组合跟踪、再平衡和绩效分析连接起来。

本项目没有复制它们的“更多图表”，而是选择一个更适合当前项目的差异化闭环：

| 同行常见能力 | 本项目本轮实现 |
|---|---|
| 筛选器/候选发现 | 已有版本化机会策略与候选漏斗 |
| 组合观察 | 不可变纸面组合与自动真实行情观察 |
| 基准比较 | A/H/美股按冻结权重合成同市场可交易基准 |
| 实际业绩 | 固定交易日、成本后净收益与净超额 |
| 策略排名 | 不按回测收益排序，按资金门禁状态和前瞻证据展示 |
| 风险控制 | 独立起点、回撤、置信区间和多重检验校正 |
| 资金建议 | 只开放受限人工试运行上限，不生成订单 |

## 3. 固定前瞻窗口

### 3.1 市场基准

当前使用可交易 ETF 作为透明近似：

| 市场 | 基准 | 说明 |
|---|---|---|
| A 股 | `510300` | 沪深 300 ETF 近似 |
| 港股 | `02800` | 盈富基金近似 |
| 美股 | `SPY` | 标普 500 ETF 近似 |

每只股票按自己的冻结日期确定股票和基准起点，再按冻结权重聚合。跨市场组合仍是各本币收益的权重近似，暂未纳入持有期汇率变化。

### 3.2 精确第 N 个真实交易日

股票冻结价为 \(P_{i,0}\)，冻结后第 \(N\) 个真实交易日复权收盘价为 \(P_{i,N}\)，冻结权重为 \(w_i\)：

\[
R_{gross,N}=\sum_i w_i\left(\frac{P_{i,N}}{P_{i,0}}-1\right)
\]

基准用同样方法计算：

\[
R_{benchmark,N}=\sum_i w_i\left(\frac{B_{i,N}}{B_{i,0}}-1\right)
\]

系统直接从截至观察日的真实历史序列中定位第 5/20/60 个交易日。即使调度器停机后在第 40 天补跑，20 日窗口仍读取第 20 个交易日，不会使用第 40 天收益。

观察载荷同时保存：

- 股票和基准的目标日期、目标价格和来源；
- 当前观察截面；
- 股票/基准覆盖权重；
- 每个窗口的精确完成状态；
- 统一成本情景；
- 保守成分回撤；
- 行情截面幂等键和事件哈希链。

## 4. 成本后收益与基准超额

若冻结投入比例为 \(W\)，往返成本情景为 \(c\) bps，则成本拖累（百分比点）为：

\[
C=W\times\frac{c}{100}
\]

其中 \(W\) 使用 0 到 1 的比例。默认 90% 投入和 30 bps 成本对应 0.27 个百分点。

\[
R_{net,N}=R_{gross,N}-C
\]

\[
R_{excess,N}=R_{net,N}-R_{benchmark,N}
\]

收益政策可创建新版本，但不能覆盖旧版本。旧记分卡继续绑定当时政策。

引擎不可下调的安全底线为：

| 项目 | 安全底线 |
|---|---:|
| 往返成本 | 不低于 10 bps |
| 股票/基准覆盖 | 不低于 80% |
| 主窗口成熟独立批次 | 不少于 6 |
| 平均成本后净超额 | 不低于 0% |
| 胜过基准批次比例 | 不低于 50% |
| 最差批次回撤上限 | 不高于 25% |
| 人工试运行上限 | 不高于组合 5% |
| 最新候选有效期 | 不超过 30 天 |

用户可以把政策调严，不能把它调到比这些底线更宽松。

## 5. 独立批次，不能重复刷样本

同一天重复运行六次不是六个独立市场试验。

每个窗口把冻结起点间隔转换为：

\[
D_N=\left\lceil N\times\frac{7}{5}\right\rceil
\]

因此：

| 窗口 | 最小冻结起点间隔 |
|---|---:|
| 5 个交易日 | 7 个自然日 |
| 20 个交易日 | 28 个自然日 |
| 60 个交易日 | 84 个自然日 |

每段间隔中按冻结时间和 ID 预先、确定性地选择最早代表批次。后续重叠批次状态改为 `excluded`，仍展示和保留审计，但不进入成熟样本数、均值、胜率、回撤或置信区间。

选择发生在查看收益结果之前，不能事后挑该段表现最好的批次。

## 6. 跨策略多重检验校正

对某个窗口的独立净超额样本 \(x_1,\dots,x_n\)，普通 95% t 区间为：

\[
\bar{x}\pm t_{n-1,0.975}\frac{s}{\sqrt{n}}
\]

但用户可能先后尝试很多策略版本。若历史上已有 \(m\) 个策略版本真正冻结过纸面组合，引擎额外使用 Bonferroni 家族错误率校正：

\[
\alpha_{strategy}=\frac{0.05}{m}
\]

\[
\bar{x}\pm t_{n-1,1-\alpha_{strategy}/2}\frac{s}{\sqrt{n}}
\]

只有普通区间和校正区间下界都大于 0，才能通过置信门禁。

`m` 来自历史数据库中 `strategy_version_id` 的去重纸面组合计数：

- 归档失败策略不会减少 `m`；
- 给策略创建新版本并重新冻结会增加 `m`；
- 同版本重复运行不会重复增加 `m`；
- 没有进入纸面验证的草稿不计入研究族。

这不能消除所有研究者自由度，但能阻止最明显的“批量试错再挑随机赢家”。

## 7. 资金资格状态机

| 状态 | 含义 | 最大资金资格 |
|---|---|---:|
| `empty` | 尚未冻结纸面组合 | 0% |
| `collecting` | 独立成熟批次不足 | 0% |
| `watch` | 点估计可能过线，但统计区间仍覆盖零超额 | 0% |
| `suspended` | 平均超额、胜率或回撤未通过 | 0% |
| `limited_manual_pilot` | 全部门禁通过 | 政策上限，硬上限 5% |

`limited_manual_pilot` 只表示可以计算研究预算，不能直接获得金额。资金计划还必须同时满足：

1. 当前用户已有完整、可核验且已激活的个人投资政策；
2. 月度预算大于 0；
3. 当前人民币组合估值通过交易金额门禁；
4. 最新纸面组合在政策有效期内；
5. 标的市场属于用户允许市场；
6. 加入计划后不超过个人单品仓位上限。

最终计划只返回人民币研究金额和继续留现金金额，不返回股数、订单或券商指令：

```json
{
  "status": "available",
  "planned_budget_cny": 5000,
  "allocated_amount_cny": 4500,
  "unallocated_cash_cny": 500,
  "execution_authorized": false
}
```

## 8. 自动观察与任务隔离

Celery Beat 注册 `observe-opportunity-baskets`，默认每小时调用：

```text
stock_assistant.scheduler.opportunity_observations
```

调度器只做以下事情：

1. 跨用户读取最小纸面组合范围；
2. 跳过完整性失败、最大窗口完成和 18 小时内已观察的组合；
3. 按 `basket_id + UTC 日期` 创建持久后台作业；
4. 把完整输入保留在 PostgreSQL，只向 Celery 发送 Job ID。

真正的股票和基准行情读取仍在 `market-data` Worker 中执行。观察载荷使用股票/基准日期、价格和精确窗口结果形成 SHA-256 幂等键；同一行情截面重复请求返回已有观察，不追加事件。

## 9. 不可变存储

PostgreSQL/SQLite 新增：

- `opportunity_profit_policy_versions`
- `opportunity_profit_scorecards`

两张表均：

- 按 `user_id` 隔离；
- 绑定策略及策略版本；
- 保存规范化 JSON 与 SHA-256；
- 拒绝 UPDATE/DELETE；
- 旧政策和旧记分卡不会被新行情覆盖。

纸面观察表新增 `idempotency_key` 与部分唯一索引：

```text
(user_id, basket_id, idempotency_key)
WHERE idempotency_key IS NOT NULL
```

生产迁移为 `opportunity-profit-engine.v1`，使用 PostgreSQL advisory transaction lock。readiness 新增 `opportunity_profit_schema`，缺表时两个 API 副本都不能把结构不完整的数据库视为可接流量。

记分卡冻结会先把默认政策写成不可变 v1。相同证据截止点、政策、统计结果、个人政策和估值上下文不会因为墙钟生成时间不同而重复写卡；绑定事实变化后才生成新卡。

## 12. 明确限制

- 本功能不能保证盈利，也不能把统计显著解释为未来一定有效。
- 内置候选池仍不是交易所历史全量成分库，幸存者偏差只能通过前瞻冻结降低，不能完全消除。
- A/H/美股混合组合暂未纳入持有期汇率变化。
- ETF 是透明可交易基准近似，不是机构总收益指数。
- 统一成本压力不是用户真实券商结算；整手、停牌、涨跌停、分红税、冲击成本和现金利息仍未建模。
- Bonferroni 能控制明显的跨版本海选偏差，但不能识别用户在创建版本前做过的所有人工尝试。
- 当前只有人工研究预算，不读取券商现金余额、不生成股数、不连接券商。
