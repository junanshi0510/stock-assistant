# 五阶段决策闭环与信息架构重构

## 目标

本次迭代不增加孤立分析工具，先解决功能入口过多、首页信息密度失控、投资准备与市场研究混在一起的问题。产品只保留一条主逻辑：先确认个人组合事实和约束，再形成研究与 Agent 结论。

## 决策闭环

后端 `investment_decision_workflow.v1` 固定返回五个有先后依赖的阶段：

1. 持仓事实：真实持仓可读取，持仓数量与金额分配完整，且每项金额大于零。
2. 投资政策：存在已激活、未到复核期且完整性通过的用户确认版本。
3. 持有纪律：每项当前持仓都存在有效且哈希校验通过的持有逻辑版本。
4. 交易账本：至少存在一笔真实交易，账本无读取错误，收益口径状态为可用。
5. 组合报告：最新报告仍绑定当前持仓、政策、账本和持有纪律，报告完整性通过，状态为 `reviewable` 或 `partial`。

阶段状态只有 `complete`、`incomplete`、`blocked`、`unavailable`。前序阶段未完成时，后序阶段只能是 `blocked`；数据读取失败时使用 `unavailable`，不生成替代状态。接口始终只选择最早的待处理阶段作为 `next_action`。

## 工作区边界

### 今日决策

- 展示当前唯一下一步、五阶段进度和风险优先的行动清单。
- 展示紧凑的资产与市场证据快照，不承载完整编辑器和大型榜单。
- 数据来源失败时保留错误范围，失败数据不参与结论。

### 我的资产

- 持仓与纪律：持仓导入、金额确认、详情和逐项退出纪律。
- 投资政策：目标、期限、预算、风险上限、市场权限、不可变版本与审计。
- 交易账本：真实现金流、费用、FIFO、XIRR 和份额核对。
- 观察清单：尚未进入组合的研究对象。

### 研究中心

- 基金研究：发现、单基金研究、比较和替代。
- 股票与板块：市场雷达、行业概念、个股、多股对比和批量筛选。
- 策略验证：使用真实历史数据检验已有信号，不把回测当成未来承诺。

### 投资 Agent

- 只在真实工具、Evidence、风险门禁和可选 LLM 网关上运行。
- 保留单基金和 2-6 只基金批量研究、运行历史、结果比较、Outcome 和审计。
- 未配置模型时明确显示未生成模型研判，不用模板文本兜底。

## 界面调整

- 顶层导航由六个功能入口收敛为四个业务工作区。
- 首页使用命令区、阶段进度和一个下一步，减少重复组合说明。
- 投资政策从首页移到独立页面，表单按三组业务问题组织。
- 研究中心新增对象级导航，避免基金、股票和回测直接争夺顶层位置。
- 390px 手机端主导航使用四等分稳定网格，表单和研究入口改为单列；没有横向溢出或控件截断。

## 后续约束

新增能力必须回答它属于哪个工作区、服务哪个决策阶段、读取什么真实 Evidence、失败时如何停止结论。不能仅因为有数据接口就增加顶层入口，也不能绕过投资政策、持有纪律或账本直接生成个性化仓位动作。
