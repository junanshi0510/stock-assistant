# 2026-07-15 更新 006：基金换仓真实现金流与执行前审查

## 1. 目标

本轮把“候选基金研究”推进到可审计的执行前复核，但不接入交易权限。

系统现在能够回答：

1. 用户从销售平台看到的本次赎回和拟申购金额是否能闭合。
2. 该报价是否仍绑定当前 FIFO 账本、当前持仓和当前成本证据。
3. 换仓后的单品、权益和行业最坏暴露是否仍满足用户激活的投资政策。
4. 候选底层市场、汇率风险和用户允许市场是否一致。
5. 哪些真实证据阻止本次赎回复核继续进行。

系统仍然不能回答或执行：

- 不预测换仓后必然盈利。
- 不代替用户判断自由文本退出条件是否已触发。
- 不发送赎回或申购订单。
- 不把提交前报价冒充最终成交或清算结果。
- 不允许用预计赎回款垫资提前申购。

## 2. 决策链路

```text
同类候选
  -> 滚动持续性
  -> 费率与披露持仓尽调
  -> 用户 FIFO 成本快照
  -> 销售平台 v2 真实现金流报价
  -> IPS / 持有逻辑 / 市场权限 / 换仓后穿透审查
  -> 人工赎回复核
  -> 等待真实到账
  -> 重新获取候选申购报价
  -> 用户人工下单
  -> 导入实际成交并核对账本
```

任何上游阶段失败都会停止后续阶段。没有“默认通过”或模板兜底。

## 3. 平台报价 v2

`fund_switch_platform_quote.v2` 新增以下强制输入：

| 字段 | 来源 | 校验 |
| --- | --- | --- |
| 赎回总额 | 销售平台本次提交前确认页 | 必须大于零 |
| 赎回费 | 销售平台确认页 | 非负且小于赎回总额 |
| 候选拟申购总额 | 用户准备在到账后提交的金额 | 必须大于零且不超过预计净到账 |
| 候选申购费 | 销售平台确认页 | 非负且小于拟申购总额 |
| 报价时间 | 销售平台确认页 | 包含时区且不超过 24 小时 |
| 预计赎回到账日 | 销售平台确认页 | 不早于报价日且不超过 30 天 |
| 当前可申购 | 销售平台确认页 | 为否时成本门禁关闭 |
| 在途风险确认 | 用户明确确认 | 必须为真 |

服务端只做确定性现金流计算：

```text
预计赎回净到账 = 平台赎回总额 - 平台赎回费
候选净投资额 = 候选拟申购总额 - 候选申购费
预计留存现金 = 预计赎回净到账 - 候选拟申购总额
```

平台赎回总额与最新确认净值估算偏差超过 `max(10 元, 估算总额的 0.5%)` 时，必须单独确认。费用与披露成本区间偏差超过 `max(10 元, 平台赎回总额的 0.2%)` 时，也必须单独确认。

旧 `fund_switch_platform_quote.v1` 记录保留在审计链中，但动态状态变为 `superseded`，原因是缺少真实现金流字段。

## 4. 执行前审查门禁

`fund_switch_execution_review@1.0.0` 执行以下门禁：

| 门禁 | 通过条件 |
| --- | --- |
| 平台报价 | v2、当前有效、现金流闭合、成本和报价审计链完整 |
| 投资政策 | 当前激活、未过期、适当性有效、治理审计链完整 |
| 持有逻辑 | 当前持仓存在有效版本，用户确认已人工阅读退出条件 |
| 候选市场 | 真实基金元数据能够识别所需市场权限 |
| 市场权限 | 候选所需市场属于 IPS 允许范围 |
| 汇率风险 | 跨境候选必须由 IPS 明确接受汇率风险 |
| 现金流 | 拟申购金额不超过预计净到账，禁止垫资 |
| 单品上限 | 换仓后候选金额占已投资资产比例不超过 IPS 上限 |
| 穿透证据 | 所有预计持仓基金的真实定期披露完整、新鲜且无冲突 |
| 权益上限 | 换仓后权益最坏上界不超过 IPS 上限 |
| 行业上限 | 换仓后任一行业最坏上界不超过 IPS 上限 |

穿透计算继续使用保守区间：未披露权益进入上界，不按基金名称猜测股票、行业或市场。任一真实披露请求失败时，审查保存失败代码和错误摘要，并以 `blocked_by_exposure_evidence` 结束。

## 5. 状态语义

| 状态 | 含义 |
| --- | --- |
| `ready_for_redemption_review` | 只允许进入用户人工赎回复核 |
| `blocked_by_quote` | 报价或现金流证据不可用 |
| `blocked_by_policy` | IPS 未配置、过期或完整性失败 |
| `blocked_by_thesis` | 持有逻辑缺失、非活动版本或用户未确认复核 |
| `blocked_by_market` | 市场识别、市场权限或汇率风险门禁失败 |
| `blocked_by_cashflow` | 金额无法闭合或候选不可申购 |
| `blocked_by_exposure_evidence` | 真实定期披露不足 |
| `blocked_by_portfolio_limit` | 单品、权益或行业上限被触发 |
| `expired` | 绑定的平台报价已过期 |
| `superseded` | 报价、持仓、IPS 或持有逻辑已变化 |
| `integrity_failed` | 载荷、前序哈希或绑定证据校验失败 |

所有状态都固定返回：

```text
candidate_purchase_ready = false
full_switch_execution_ready = false
execution_authorized = false
automatic_redemption_allowed = false
automatic_purchase_allowed = false
```

## 6. 审计与不可变存储

新增 `fund_switch_execution_reviews`：

- 按用户、持仓和候选基金维护单调递增 revision。
- 每条记录绑定报价事件 ID 与事件哈希。
- 同时绑定成本快照、IPS、持有逻辑、当前持仓、预计持仓、市场画像、穿透结果和逐基金披露 SHA-256。
- 载荷 SHA-256、证据 SHA-256、前序审查哈希和本次审查哈希分别保存。
- SQLite 触发器禁止 `UPDATE` 和 `DELETE`。
- 查询时重新检查当前报价、当前持仓、当前 IPS 和当前持有逻辑；变化只改变动态状态，不覆盖历史载荷。

缺少 IPS 或持有逻辑时，“缺失”本身也会被绑定为当前事实。以后新建版本时，旧审查才会变为 `superseded`，不会在刚生成时误报为旧记录。

## 7. Agent 边界

Agent 私有上下文只暴露：

- 候选代码和当前审查状态。
- 阻断项。
- 精简现金流摘要。
- 换仓后单品、权益和行业上限摘要。
- 审计完整性和当前绑定状态。

不暴露持有逻辑自由文本、平台名称、原始 FIFO 批次或完整穿透明细。提示词版本升级到 `fund_decision_synthesis@1.9.0`，模型只有在审查状态和全部当前绑定同时通过时，才可使用“可进入人工赎回复核”；不得使用“申购可执行”“整笔换仓已就绪”或自动交易表述。

## 8. API 与界面

新增：

```text
POST /api/holdings/{holding_id}/fund-switch-execution-reviews/{candidate_code}
GET  /api/holdings/{holding_id}/fund-switch-execution-reviews/{candidate_code}
```

持仓详情的每个候选行按固定顺序展示：

1. FIFO 披露成本。
2. 销售平台真实现金流。
3. 组合与政策执行审查。
4. 人工赎回、等待到账、申购重报和成交核对四个阶段。

界面不新增顶层模块，也不把审查结果做成买卖按钮。

## 9. 监管与产品依据

- 基金费用会降低投资回报，赎回费和购买费可能因产品与平台而异：[Investor.gov 基金和 ETF 费用说明](https://www.investor.gov/introduction-investing/general-resources/news-alerts/alerts-bulletins/investor-bulletins/mutual-fund-and-etf-fees-and-expenses-investor-bulletin)。
- 开放式基金通常按订单提交后计算的下一次净值成交，提交时未必知道最终金额：[SEC Mutual Fund Guide](https://www.sec.gov/investor/pubs/sec-guide-to-mutual-funds.pdf?force_download=true)。
- 中国公募基金销售费用规则已经修订并自 2026 年 1 月 1 日施行，平台报价不能长期缓存为固定规则：[中国证券投资基金业协会通知](https://www.amac.org.cn/xwfb/zjyw/202601/t20260106_27176.html)。

因此，本版本明确区分“披露费率”“提交前平台报价”“最终成交清算”三个口径。

## 11. 下一阶段

下一阶段应实现“真实到账与申购重报价闭环”，而不是直接接自动交易：

1. 用户导入实际赎回成交与到账记录。
2. 系统核对实际净到账与 v2 预计值的差异。
3. 到账后获取新的候选申购平台报价。
4. 重新执行 IPS、仓位和披露新鲜度门禁。
5. 用户人工成交后导入实际份额和费用。
6. 将预计、实际和后续收益归因分开保存，避免把未执行建议计入 Agent 盈利。
