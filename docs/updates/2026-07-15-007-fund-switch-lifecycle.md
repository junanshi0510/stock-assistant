# 2026-07-15 更新 007：真实基金替换批次与历史收益归因

## 1. 目标与边界

本轮把基金替代链路从“执行前复核”推进为可审计的执行后闭环，回答四个不同问题：

1. 赎回是否真的成交并到账，金额是否与交易流水一致。
2. 到账后重新看到的候选申购条件是否仍满足当前投资政策和组合上限。
3. 用户已经完成的候选申购是否与到账资金、平台重报价和当前持仓一致。
4. 截至一个真实确认净值日，已经发生的替换相对“继续持有原基金”历史上多赚或少赚了多少。

本功能不发送订单、不代替用户登录销售平台、不预测替换后必然盈利，也不把执行前报价、盘中估值或历史相对结果解释为未来收益。

## 2. 完整决策链

```text
同类候选与持续性门禁
  -> 费率和定期披露尽调
  -> 用户 FIFO 成本快照
  -> 销售平台执行前真实现金流报价
  -> IPS / 持有逻辑 / 市场 / 换仓后穿透审查
  -> 用户自行提交赎回
  -> 导入真实卖出流水并确认实际到账
  -> 到账后重新录入申购金额、费用、限额和确认日
  -> 重新执行 IPS / 市场 / 汇率 / 仓位 / 穿透门禁
  -> 用户自行提交申购
  -> 导入真实买入流水
  -> 当前确认持仓与 FIFO 账本对账
  -> 使用两只基金同日确认净值做历史反事实归因
```

任何上游事实缺失都会停止后续阶段。系统没有默认通过、模板结论或模拟数据兜底。

## 3. 状态机

`fund_switch_lifecycle@1.0.0` 使用用户级追加式事件流：

| 顺序 | 事件 | 可重复 | 核心证据 |
| --- | --- | --- | --- |
| 1 | `redemption_settled` | 否 | 执行审查、卖出流水、提交时间、到账日和实际到账金额 |
| 2 | `purchase_requoted` | 是，成交前可更新 | 到账后平台页面、当前 IPS、当前持仓、市场画像和组合穿透 |
| 3 | `purchase_recorded` | 否 | 最新有效重报价、真实买入流水和人工提交时间 |
| 4 | `holdings_reconciled` | 否 | 原基金与候选基金当前确认份额、FIFO 剩余批次和完整性问题 |
| 5 | `attribution_snapshot` | 是，对账后可刷新 | 两只基金确认净值、分红送配和绑定交易快照 |

动态状态包括：

| 状态 | 含义 |
| --- | --- |
| `settled_purchase_requote_required` | 赎回已真实到账，必须重新确认候选申购条件 |
| `purchase_requote_blocked` | 到账后现金、政策、市场或组合门禁未通过 |
| `purchase_requote_expired` | 到账后申购报价超过 24 小时 |
| `purchase_requote_superseded` | 报价后 IPS 或持仓发生变化 |
| `ready_for_manual_purchase_review` | 只允许用户人工复核并自行操作，不是下单授权 |
| `purchase_recorded_reconciliation_pending` | 已绑定真实申购流水，等待当前持仓对账 |
| `completed_attribution_pending` | 事实已对账，尚未生成历史归因 |
| `completed_attribution_available` | 同日真实净值历史归因可用 |
| `completed_attribution_blocked` | 归因所需真实数据不完整或口径冲突 |
| `integrity_failed` | 事件链、执行审查或绑定交易流水已变化 |

## 4. 真实赎回与到账

启动批次必须同时满足：

- 执行前审查状态仍为 `ready_for_redemption_review`，审查哈希和当前绑定完整。
- 赎回提交时间位于执行前平台报价有效期内，且不能晚于当前时间。
- 卖出流水属于当前基金，份额覆盖成本快照绑定的全部确认份额。
- 从账本移除本次卖出后，原 FIFO 批次哈希仍与执行前成本快照一致，证明期间没有未复核的账本变化。
- 实际到账日期不早于成交确认日，实际到账金额等于 `成交份额 × 确认净值 - 赎回费`，误差不超过 0.02 元。
- 实际赎回总额明显偏离执行前报价时，用户必须显式确认已经复核差异。

同一卖出流水只能绑定一个替换批次。并发重复提交由数据库唯一索引和事务共同拒绝。

## 5. 到账后申购重报价

系统不复用执行前候选报价。到账后必须重新录入：

- 销售平台名称和报价时间。
- 候选申购总额与实际申购费。
- 预计确认日期和当前可申购状态。
- 用户确认这些字段来自到账后的销售平台页面。

确定性门禁重新检查：

| 门禁 | 通过条件 |
| --- | --- |
| 真实到账现金 | 申购总额不超过实际到账金额，不允许预计现金垫资 |
| 当前投资政策 | 激活、未过期、适当性和治理审计完整 |
| 候选市场 | 真实基金元数据可识别所需市场 |
| 市场与汇率权限 | 所需市场在 IPS 允许范围，跨境基金已确认汇率风险 |
| 单品仓位 | 申购后候选占比不超过单品上限 |
| 组合穿透 | 全部基金真实定期披露完整、新鲜、无冲突且绑定当前投影 |
| 权益和行业上限 | 最坏上界均不超过 IPS 限制 |

报价保存后 24 小时失效。报价后持仓或 IPS 变化会把状态改为 `purchase_requote_superseded`，但旧事件仍保留。

## 6. 实际申购与持仓对账

申购记录必须绑定最新一条有效 `purchase_requoted` 事件，并满足：

- 买入流水属于候选基金且确认日期不早于实际到账日。
- 人工提交时间位于到账后报价有效期内。
- `份额 × 确认净值 + 申购费` 不超过实际到账现金。
- 实际使用现金明显偏离报价时必须显式确认差异。
- 同一买入流水只能绑定一个替换批次。

申购回填后不会立即标记完成。持仓对账还要求：

- 原基金 FIFO 剩余份额为零。
- 当前确认持仓不再包含原基金份额，或其确认份额为零。
- 候选基金当前确认份额与 FIFO 账本剩余份额一致。
- 两只基金的交易流水没有超卖或其他份额完整性问题。

## 7. 历史反事实归因

归因只在以下条件全部满足时生成：

- 原基金赎回确认日存在来源确认净值。
- 候选基金申购确认日存在来源确认净值。
- 交易流水确认净值与来源同日净值误差不超过 0.2%。
- 两只基金在成交后存在一个共同的真实确认净值日。
- 比较区间没有未进入交易账本的分红或拆分。
- 净值和分红送配来源均成功返回，不使用估算净值、近似日期或旧缓存代替。

计算口径：

```text
实际替换路径价值 = 候选确认份额 × 候选截止净值 + 剩余现金
继续持有反事实价值 = 原基金赎回份额 × 原基金截止净值
相对增量价值 = 实际替换路径价值 - 继续持有反事实价值
相对增量回报 = 相对增量价值 / 原基金实际赎回总额
实际总费用 = 实际赎回费 + 实际申购费
```

该结果只描述已发生决策截至历史时点的相对表现。它不是未来上涨概率、目标收益或新的买卖信号。

## 8. 不可变存储与完整性

新增 `fund_switch_lifecycle_events`：

- 每条事件保存用户、批次、持仓、原基金、候选基金和执行审查身份。
- 保存事件序号、事件类型、状态、载荷 SHA-256、证据 SHA-256、前序事件哈希和本次事件哈希。
- 赎回和申购事件同时保存绑定交易 ID 与交易快照 SHA-256。
- 数据库触发器禁止 `UPDATE` 和 `DELETE` 生命周期事件。
- 部分唯一索引阻止同一真实赎回或申购流水被多个批次重复使用。
- 查询时重新计算事件链并读取当前交易快照；交易被删除或变化后动态返回 `integrity_failed`。

交易流水本身仍允许用户纠错或删除，但此操作不会伪造连续历史，而是显式使相关替换批次失效。

持仓详情继续使用原有候选行，不新增顶层功能入口。批次启动后执行前报价锁定，后续只追加真实执行事实。

“我的资产 → 交易与复盘”新增全量批次表并可展开同一操作面板。即使赎回后原基金已经从当前持仓删除，用户仍能继续重报价、回填申购、对账和归因。

## 10. Agent 最小化边界

私有组合上下文只向模型发送：

- 原基金代码、候选代码和当前阶段。
- 实际到账、实际申购和历史归因的必要聚合数字。
- 当前门禁、完整性和是否已对账。

不发送销售平台、交易流水 ID、批次 ID、事件哈希或原始持仓明细。提示词版本升级为 `fund_decision_synthesis@2.0.0`：

- 只有到账后全部门禁当前有效时，模型才可称“真实到账后的申购门禁已通过，可由用户人工复核”。
- 不得称“建议买入”“已授权申购”或暗示系统会自动交易。
- 归因必须称为已发生决策的历史结果，不得外推为未来盈利能力。

## 11. 监管和产品依据

- 中国证监会现行公募基金销售费用规则强调费用安排和投资者权益保护，具体交易仍应以基金合同、销售文件和销售平台确认结果为准：[《公开募集证券投资基金销售费用管理规定》](https://www.csrc.gov.cn/csrc/c101954/c7606091/content.shtml)。
- 基金销售机构应确保结算资金安全、及时划付，并向投资者提供确认日期、份额和金额等交易确认信息：[《公开募集证券投资基金销售机构监督管理办法》](https://www.csrc.gov.cn/csrc/c106256/c1653806/content.shtml)。
- 开放式基金交易通常采用订单提交后计算的下一次净值，提交前报价与最终确认净值必须分开：[SEC Mutual Fund Guide](https://www.sec.gov/investor/pubs/sec-guide-to-mutual-funds.pdf?force_download=true)。

因此，本实现明确分离披露费率、执行前平台报价、实际清算、到账后重报价、最终申购成交和后续历史归因。
