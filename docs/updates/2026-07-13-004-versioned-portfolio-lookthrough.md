# 2026-07-13 更新 004：版本化基金穿透与组合约束执行

## 1. 更新结论

本轮新增“版本化组合穿透快照”，并将其接入基金个性化 Agent 决策。

系统现在不会因为目标基金本身没有超过单品仓位上限，就直接给出新增金额。每次个性化基金研究还必须用用户确认持仓、基金真实资产配置和定期报告披露形成不可变快照，验证：

- 组合权益总仓位是否超过已激活 IPS 的上限。
- 同一行业通过多只基金叠加后是否超过单行业上限。
- 未披露或无法分类的权益是否可能在最坏情况下导致超限。
- ETF 联接基金是否能够通过定期报告验证目标 ETF 及其真实占净值比例后继续穿透。
- 快照是否新鲜、完整、哈希一致，并与当前 Run 的持仓和 IPS 版本一致。

已证明超限时阻断新增风险；已披露下界未超限但最坏上界可能超限时，Agent 弃权并等待更完整披露；只有最坏上界仍符合约束时，才允许进入金额计算。

## 2. 为什么必须做

### 2.1 更新前的真实缺口

更新前已经存在 `/api/holdings/exposure`，可以临时汇总最多 10 只基金的披露股票和行业，但它存在五个决策缺口：

1. 结果只在请求时计算，没有不可变快照、内容哈希和历史版本。
2. 默认只加载金额最高的有限基金，不能证明整个组合符合上限。
3. 没有披露期限门禁，过期报告可能被当作当前组合依据。
4. 未披露仓位没有进入风险上界，容易把“没有看到”误解为“没有风险”。
5. 个性化 Agent 只执行单只基金上限，没有执行 IPS 已保存的权益和行业上限。

因此，用户同时持有多只不同名称、但底层集中于同一行业或同一市场的基金时，旧策略仍可能给出新增金额。这个风险直接影响投资决策，所以本轮功能是必要能力，不是展示增强。

### 2.2 为什么不先做更多预测模型

在组合约束没有执行前增加预测模型，会放大错误金额的风险。即使方向判断正确，仓位过度集中也可能让组合回撤超过用户承受能力。

本轮优先级高于继续增加预测信号，原因是：

- 风险边界是任何收益策略的前置条件。
- 组合集中风险可以用真实持仓和披露确定性计算。
- 预测模型不能替代用户确认的风险上限。
- 先形成版本化事实，后续策略评测才具有可复现输入。

### 2.3 没有采用的方案

未采用以下看似更快、但不符合工业要求的方式：

- 不按基金名称猜测行业或底层市场。
- 不把前十大持仓合计当作基金完整股票仓位。
- 不用同类平均持仓补齐披露失败的基金。
- 不把旧披露静默当作当前真实持仓。
- 不在来源失败时返回模拟行业比例。
- 不只做前端图表而不进入 Agent 风险门禁。

## 3. 数据语义

### 3.1 真实来源

快照只使用：

| 数据 | 来源 | 用途 |
|---|---|---|
| 用户持仓金额 | 用户确认保存的持仓 | 计算每项资产对总组合的贡献 |
| 股票/债券/现金占比 | 东方财富基金详情页资产配置 | 形成基金权益暴露 |
| 披露股票 | 天天基金/东方财富基金定期报告 | 形成底层证券和市场下界 |
| 行业配置 | 天天基金/东方财富基金定期报告 | 形成行业暴露下界 |
| ETF 联接关系 | 东方财富基金详情页“相关 ETF” | 取得候选目标 ETF 代码 |
| 联接基金目标 ETF 占比 | 东方财富基金公告列表与定期报告正文 | 验证目标代码并取得占净值比例 |
| 报告期 | 同一资产配置和定期报告 | 执行数据期限门禁 |
| IPS 版本 | 已激活投资政策版本 | 读取权益和行业上限 |

### 3.2 披露不是实时持仓

定期报告通常滞后。快照明确保存资产配置期、股票披露期和行业披露期，不把它们描述为实时持仓。

默认决策期限是 200 天，可通过 `PORTFOLIO_EXPOSURE_MAX_AGE_DAYS` 配置，允许范围为 30 至 730 天。任一参与决策的权益或行业披露缺少可解析报告期、或超过期限，快照不具备放行金额资格。

零权益基金是例外：真实资产配置已经证明股票占比为 0 时，不强制要求行业披露，因为不存在需要分类的权益仓位。

## 4. 区间风险模型

### 4.1 权益区间

对每只基金：

- 存在有效资产配置股票占比时，上下界都等于该真实披露值。
- 缺少资产配置时，已披露股票占比合计作为下界，100% 作为最坏上界。
- 直接股票持仓按 100% 权益处理。
- 无法识别的其他资产不猜测，下界为 0、上界为 100%。

组合权益区间：

```text
权益下界金额 = Σ(持仓金额 × 单项权益下界)
权益上界金额 = Σ(持仓金额 × 单项权益上界)
权益区间占比 = 权益区间金额 ÷ 用户确认组合总金额
```

### 4.2 行业区间

每个已披露行业形成可证明下界。基金权益上界中没有被行业披露解释的部分，进入“未分类权益”。

对任一行业的最坏上界：

```text
行业最坏上界金额 = 该行业已披露金额 + 全部未分类权益金额
```

这代表未分类权益可能全部集中于任一行业。该计算是保守风险边界，不是假设真实持仓一定如此。

### 4.3 市场区间

底层证券仅在标识符足够明确时归类：

- A 股：明确的 6 位 A 股代码段。
- 港股：5 位数字证券代码。
- 美股：可识别的英文字母证券代码。
- 其他标识符：保留为 `unknown`。

直接股票优先使用用户确认市场；无法识别时才检查证券代码。基金名称不会用于填充底层市场。

当前市场区间用于展示和 Evidence。基金是否允许投资于内地、港股、美股或全球市场，仍由已上线的基金市场画像和 IPS 市场权限门禁执行。

### 4.4 ETF 联接基金两层穿透

ETF 联接基金自身通常不直接披露股票，原股票持仓接口可能返回空数据。本轮增加通用的一层联接穿透：

1. 从基金详情页读取“查看相关 ETF”链接，取得目标 ETF 候选代码。
2. 从基金公告 API 读取最新定期报告列表。
3. 从定期报告 `2.1.1 目标基金基本情况`读取目标基金主代码。
4. 要求详情页代码与定期报告代码完全一致，否则拒绝穿透。
5. 从定期报告 `5.9` 节读取目标 ETF 占基金资产净值比例。
6. 读取目标 ETF 同期真实资产配置、股票和行业披露。
7. 按联接基金真实目标 ETF 占比逐项缩放。

```text
联接基金底层占比 =
  联接基金持有目标 ETF 占净值比例
  × 目标 ETF 对应资产/股票/行业披露比例
```

关系、报告代码、报告期、公告链接、PDF 链接、目标 ETF 报告期和缩放前占比全部进入 Evidence。代码不一致、报告章节缺失或目标 ETF 真实披露不可用时，不使用名称匹配或硬编码映射兜底。

## 5. 不可变快照

### 5.1 数据表

新增 `portfolio_exposure_snapshots`：

| 字段 | 说明 |
|---|---|
| `id` | 快照 ID |
| `user_id` | 用户范围，当前迁移阶段为 `default` |
| `schema_version` | `portfolio_exposure_snapshot.v1` |
| `holdings_sha256` | 本轮用户确认持仓规范化哈希 |
| `target_code` | 本轮研究目标基金 |
| `profile_version_id` | 本轮绑定 IPS 版本 |
| `status` | `complete`、`partial` 或 `unavailable` |
| `payload_json` | 完整区间、来源、报告期和质量门禁 |
| `payload_sha256` | 规范化载荷 SHA-256 |
| `created_at` | 持久化时间 |

同一用户相同内容哈希只保存一次，重复刷新幂等返回已有快照。

### 5.2 不可变性

数据库触发器禁止更新任何已保存快照。快照变化必须生成新记录，不能覆盖历史输入。

`verify_portfolio_exposure_snapshot` 会重新计算载荷哈希，并交叉校验：

- 持仓哈希。
- Schema 版本。
- 目标基金代码。
- IPS 版本。
- 持久化载荷 SHA-256。

## 6. 数据质量门禁

快照只有同时满足以下条件才为 `decision_eligible=true`：

1. 至少存在一项用户确认持仓。
2. 每项持仓金额有效，组合总金额大于 0。
3. 所有金额大于 0 的六位基金代码都成功取得真实披露。
4. 没有资产配置与股票/行业披露数值冲突。
5. 需要权益或行业判断的报告期均可解析且未过期。
6. 目标基金真实披露可用。
7. 目标基金披露未过期且没有数值冲突。
8. 快照持久化哈希验证通过。
9. 快照持仓哈希与 Agent 组合上下文一致。
10. 快照 IPS 版本与 Agent Run 固定版本一致。

来源失败、报告期缺失、数值冲突和目标披露不可用都会原样进入质量原因；系统不调用模拟兜底。

## 7. Agent 工具与审计链

### 7.1 新工具

新增：

```text
portfolio.exposure.snapshot@1.0.0
```

- 风险等级：`R1`。
- 超时：120 秒。
- 输入：目标基金代码和 Run 固定的 IPS 版本。
- 输出：不可变快照、快照哈希和完整性验证结果。

外部抓取/区间计算与数据库持久化分为两个阶段。超时线程只允许计算；只有工作流确认工具未超时、Run 未取消后，才在主工作流中保存快照并写 Evidence。这样超时请求即使晚到返回，也不能产生没有 Run 绑定的孤立快照。

基金工作流顺序现在包含：

1. 基金核心分析。
2. 基金市场画像。
3. 用户组合上下文。
4. 组合穿透快照。
5. 可选估值、披露变化和替代品。
6. 个性化决策策略。

### 7.2 Run 固定绑定

`agent_runs` 新增 `exposure_snapshot_id`。

工作流生成快照后：

- 必须验证快照完整性和载荷哈希。
- 每个 Run 只能绑定一个快照。
- 重复绑定同一个 ID 幂等。
- 尝试绑定不同 ID 会失败。
- 审计链追加 `run.exposure_snapshot_bound`。

Run 完整性验证现在交叉检查：

- `agent_runs.exposure_snapshot_id`。
- `portfolio_exposure` Evidence 中的快照 ID。
- 快照哈希和内部完整性结果。
- `run.exposure_snapshot_bound` 审计事件。

任一项不一致，Run Evidence 完整性返回 `exposure_snapshot_binding_mismatch`。

### 7.3 重跑对比

父子 Run 对比新增“组合穿透快照”维度。持仓或披露发生变化时，重跑结果会明确标记所用快照不同，不把策略差异错误归因于行情变化。

## 8. 个性化决策策略 1.2.0

`personalized_fund_decision` 从 `1.1.0` 升级为 `1.2.0`。

### 8.1 新门禁

新增三项门禁：

- `portfolio_exposure_snapshot`：快照完整性、持仓哈希和 IPS 绑定。
- `equity_exposure_limit`：组合权益上下界与 IPS 权益上限。
- `industry_exposure_limit`：组合行业集中上下界与 IPS 单行业上限。

### 8.2 决策规则

| 状态 | 动作 |
|---|---|
| 快照缺失、过期、冲突或绑定失败 | `exposure_data_required`，不生成金额 |
| 权益或行业已披露下界已经超限 | `reduce_exposure`，不新增风险 |
| 下界未超限、最坏上界可能超限 | `exposure_data_required`，等待更完整披露 |
| 最坏上界未超限 | 继续执行市场、风险、期限、回撤和历史条件门禁 |

### 8.3 新增资金上限

对于目标基金会增加的风险维度，按最坏上界计算最大新增金额：

```text
最大新增金额 =
  (IPS 上限 × 当前组合总金额 - 当前最坏暴露金额)
  ÷ (目标基金最坏暴露比例 - IPS 上限)
```

最终允许研究金额取以下最小值：

- 用户本轮计划金额或月度预算。
- 单只基金仓位最大新增金额。
- 权益总仓位最大新增金额。
- 每个可能行业的最大新增金额。

若目标基金最坏暴露比例不高于 IPS 上限，该维度不会单独限制新增金额；其他门禁仍然有效。

## 13. 安全与隐私边界

当前生产仍是单用户迁移架构，快照 API 尚未具备正式多租户身份隔离。因此：

- 本轮没有开放自动交易或外部券商写操作。
- Agent 工具仍限制为 R0/R1。
- 快照只保存规范化持仓金额和披露结果，不保存 OCR 原图。
- 生产日志不应记录完整快照载荷。
- 正式多用户开放前仍必须完成域名、HTTPS、认证、授权和数据库行级隔离。

## 14. 已知边界

- 定期报告不是实时持仓，系统只能通过期限门禁和区间上界控制误用。
- 已支持由详情页关系和定期报告共同验证的一层 ETF 联接基金穿透。
- 普通 FOF、多层基金嵌套或无法取得真实基金投资明细的产品，当前仍不能继续穿透；未解释权益进入上界。
- 不同市场或基金管理人的行业分类口径可能不同，当前不做未经证实的行业语义合并。
- 无法识别的证券代码保留为未知，不按名称猜测。
- 直接股票当前没有真实行业主数据时，其权益进入未分类行业上界，可能导致 Agent 保守弃权。
- 该能力用于控制风险和提高决策一致性，不能保证盈利。
