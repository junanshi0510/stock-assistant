# 2026-07-15：可审计的投资任务收件箱

## 问题

“今日决策”原先每次刷新都会重新生成行动清单，但用户无法确认已读、延后处理或查看条件何时真正消失。把列表直接当待办会导致三个问题：刷新后状态丢失、用户可以误把风险手工标记为解决、不同用户的处理记录无法审计。

## 本次实现

- 确定性决策规则继续负责从真实持仓、投资政策、交易账本和市场证据中生成行动。
- 每个行动按 `user_id + action_key` 同步为持久任务，保存最新证据摘要、来源、优先级和稳定条件指纹。
- 用户只能执行“确认已知晓”“稍后处理”“重新打开”。任务的 `resolved` 状态只能由下一次真实数据刷新在触发条件消失后自动写入。
- 高优先级任务最多延后 24 小时，中优先级最多 72 小时，普通研究任务最多 168 小时；到期后自动重新进入待处理。
- 条件语义、优先级或来源变化会自动创建新 revision，并重新打开已经确认的任务。单纯行情数值刷新不会每天制造新任务。
- 所有创建、变化、确认、延后、到期、自动解决和重新触发事件写入追加式 SHA-256 哈希链，事件表禁止更新和删除。
- API 固定按登录主体隔离，返回结果不暴露内部 `user_id`、条件指纹或真实操作者标识。

## API

- `GET /api/decision-tasks`
- `PATCH /api/decision-tasks/{task_id}`
- `GET /api/decision-tasks/{task_id}/audit`

## 前端

任务收件箱仍位于“今日决策”，不新增一级导航。页面支持：

- 查看待处理、稍后、已确认和自动解决数量。
- 仅筛选待处理事项。
- 对当前行动执行确认、延后 24 小时和重新打开。
- 按需读取最近自动解决记录。

确认只表示用户已知晓。只有触发条件不再出现在最新真实证据中，任务才显示为已解决。
