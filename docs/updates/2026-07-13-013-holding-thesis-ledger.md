# 版本化持有逻辑与退出纪律

日期：2026-07-13

## 目标

把“为什么买、准备持有多久、什么证据要求重新评估、何时允许新增、什么情况下退出”变成每只真实持仓的明确计划，并用用户确认持仓与真实基金净值定期复核。

本功能不预测必然上涨，也不把亏损阈值直接转换为卖出指令。它的可验证目标是减少没有计划的追涨、亏损后的情绪化补仓、盈利后的无意集中，以及原逻辑失效后继续机械持有。

Investor.gov 将投资目标、时间期限、风险承受能力、资产配置和定期再平衡列为投资计划的重要组成部分：

- [Introduction to Investing](https://www.investor.gov/introduction-investing)
- [Asset Allocation and Diversification](https://www.investor.gov/introduction-investing/getting-started/asset-allocation)

## 用户确认字段

每只基金或股票保存：

- 组合角色：核心增长、卫星增强、防守稳定、现金流、分散风险或阶段机会。
- 买入与持有逻辑。
- 计划持有月数。
- 下次复核日期。
- 最大可接受持仓亏损比例。
- 最大可接受标的回撤比例。
- 允许新增的前提。
- 需要退出或降低仓位的条件。

系统不提供隐式阈值。用户必须逐项确认，避免把产品默认值冒充个人风险纪律。

## 不可变版本链

新增表 `holding_thesis_versions`：

| 字段 | 作用 |
|---|---|
| `id` | 唯一版本 ID |
| `asset_type / market / code` | 资产身份 |
| `version_no` | 资产内连续版本号 |
| `schema_version` | `holding_thesis.v1` |
| `state` | 当前为 `active`，预留归档事件 |
| `payload_json / payload_sha256` | 规范化内容及 SHA-256 |
| `previous_version_id` | 前一版本 ID |
| `previous_payload_sha256` | 前一版本内容哈希 |
| `created_at` | UTC 创建时间 |

SQLite 触发器拒绝 `UPDATE` 和 `DELETE`。修订只能追加新版本，验证函数逐版本检查编号连续性、内容哈希、资产身份和前序哈希。

持有逻辑同时绑定 `holding_id` 与持仓创建时间。用户删除持仓后重新添加相同代码时，旧记录只保留为历史，不进入当前计划覆盖率或行动报告。

## 真实证据复核

机器只核验结构化边界：

1. 当前持仓收益率来自用户确认持仓。
2. 基金当前回撤来自真实确认净值序列。
3. 当前日期与用户确认的复核日期比较。
4. 组合角色、期限和阈值来自最新且哈希通过的持有逻辑版本。

复核状态：

- `missing`：当前持仓没有有效计划。
- `on_track`：尚未到复核日，也未触发结构化风险边界。
- `review_due`：到达用户设定的复核日期。
- `risk_limit_breached`：持仓亏损或标的回撤触及用户预设边界。
- `unavailable`：版本完整性或字段校验失败。

加仓与退出条件是自由文本，始终返回 `machine_verified=false`。系统展示它们供用户逐项核对，不通过关键词匹配伪造触发结果。

## 行动报告绑定

组合行动报告升级为：

- Schema：`portfolio_action_report.v2`
- Ruleset：`portfolio_action_rules.v2`
- 新增绑定：`theses_sha256`

报告只对当前持仓匹配的最新计划生成确定性快照。以下任一变化都会使旧报告 `binding.current=false`：

- 持仓事实变化。
- 已激活投资政策变化。
- 当前持有逻辑版本变化。
- 报告 Schema 或 Ruleset 升级。
- 报告载荷完整性失败。

行动顺序为：数据完整性、个人风险边界、单品超限、重复暴露、预设持有纪律、回撤复核、保持纪律。单品超限和重复暴露仍优先于单资产持有逻辑，避免局部计划绕过组合风险。

## API

```http
GET /api/portfolio/theses
POST /api/portfolio/theses
GET /api/portfolio/theses/{asset_type}/{code}?market=基金&limit=20
```

列表接口返回当前持仓覆盖率。保存接口只接受已存在且市场身份完全匹配的持仓。单资产接口返回最新版本、版本历史与哈希链验证结果。

## 前端

- 不增加新导航页。
- 点击任一持仓行，在原详情抽屉中查看或修订持有逻辑。
- 新计划不预填金融阈值，必须由用户确认。
- 展示最新版本号、内容哈希、结构化复核状态和真实触发证据。
- 保存后重新读取最新计划和行动报告；旧报告失效时停止展示旧操作状态。
- 桌面端使用双列表单，手机端切换单列，抽屉宽度不超过视口。

## 失败策略

- 持仓不存在：拒绝保存。
- 日期、比例、期限或文本长度无效：拒绝保存。
- 内容哈希或版本链失败：停止使用计划边界。
- 基金真实净值不可用：不检查基金回撤边界，行动报告沿用真实来源失败门禁。
- 自由文本条件：绝不自动判定为已满足。
- 所有状态都不触发自动申购、赎回或下单。

## 测试范围

- 不可变更新和删除触发器。
- 连续版本号与前序哈希；链验证读取完整历史，不受 100 条接口分页上限影响。
- 当前持仓覆盖率。
- 删除并重建同代码持仓不会继承旧计划。
- 用户确认亏损与真实基金回撤双边界复核。
- 自由文本条件保持人工核对。
- 计划缺失进入行动顺序。
- 版本库不可用与用户未填写计划严格区分，不把系统故障归因给用户。
- 计划变化使旧行动报告失效。
- API 路由契约、前端生产构建、桌面和手机布局。
