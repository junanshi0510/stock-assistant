# 金融投资助手

一个面向个人投资者的真实数据决策工作台，覆盖公募基金、A 股、港股、美股和用户真实持仓。

当前版本使用 **React + Vite** 构建前端，使用 **FastAPI + Python** 完成数据获取、基金与股票研究、组合账本、确定性计算和证据约束的大模型合成。项目正在按照工业级投资 Agent PRD 逐步升级；当前已经具备服务端登录、管理员/用户 RBAC、个人数据隔离、可恢复 Run、真实工具、Evidence、风险门禁和可选 LLM 研判，但仍不具备自动交易能力，也不把模型文本当作确定性计算结果。

> 风险提示：系统输出用于研究和风险复盘，不代表未来涨跌，不构成投资建议，也不承诺收益。数据源不可用、数据过期或用户持仓不完整时，系统必须明确显示缺口，不使用模拟数据补齐。

### 2026-07-23：市场状态、策略适配与动态风险预算中枢

- 机会工厂新增“市场状态”大功能，把原来散落在单次扫描里的 A 股、港股、美股候选池状态提升为正式决策层，形成“不可变扫描共识 → 当前市场状态 → 同环境策略验证 → 风险预算 → 投资委员会 → 全组合资金计划”的完整链路。它回答的是“当前环境更适合哪些已经验证过的策略、原委员会限额还应保留多少”，不是把技术规则包装成某只股票的上涨概率。
- 每个市场最多读取 5 个策略版本在 14 天内的最新、哈希通过扫描，按候选样本数和 UTC 日期新鲜度加权，综合三月收益中位数、上涨广度和候选池年化波动，输出 `risk_on / mixed / defensive / insufficient`。单一来源、状态分歧、高波动和过期来源都只会降低风险预算；偏强环境最高也只是保留原始上限，永远不加杠杆。
- 新纸面组合会冻结当时的候选池市场状态；旧组合只允许从其 `run_result_sha256` 精确绑定、完整性通过的原扫描结果还原，不修改历史。策略适配只比较“冻结时与当前处于同类环境”的独立前瞻批次：少于 4 个不倾斜，最近 3 个同环境批次均未跑赢基准时熔断，负均值或低于 50% 跑赢比例时降权；只有平均超额、60% 跑赢比例和 95% 区间下界同时为正时才允许最高 10% 的窄幅优先，并按 8 个样本向中性收缩。
- 投资委员会升级为 `adaptive_strategy_committee@1.1.0`：环境失配可停用原本合格的策略，适配度参与策略袖套相对权重，当前状态和波动形成总风险硬上限。全组合资金引擎升级为 `whole_portfolio_next_best_action.v3` 并继承该上限；状态层只能增加现金，不能绕过前瞻记分卡、IPS、可信估值、已有仓位、集中度或压力情景门禁。
- 新增 4 个受认证操作、`opportunity_regime_snapshots` 用户隔离不可变表、Evidence/Result 双 SHA-256、同证据内容寻址复用、SQLite/PostgreSQL UPDATE/DELETE 拒绝触发器、`opportunity-regime-allocation.v1` 迁移和 `opportunity_regime_schema` readiness 门禁。当前 OpenAPI 共 `182` 个操作、`156` 条路径，其中机会工厂为 `23` 条路径、`28` 个操作。
- 后端全量回归 `542 tests` 通过，前端 `npm audit` 为 `0 vulnerabilities`，Vite 生产构建完成 `1853` 个模块转换。本地真实浏览器验证了三市场状态卡、候选池边界、策略适配矩阵、缺失值、投资政策未配置时使用最新冻结纸面组合做研究暴露回退、首次冻结、同日重复冻结严格幂等、历史绑定、委员会状态层接入、`390×844` 无全局横向溢出和干净会话 0 条控制台告警/错误；浏览器验收发现并修复了“按秒计算来源年龄导致证据哈希漂移”的问题。

### 2026-07-23：自适应策略投资委员会与候选共识模型组合

- 机会工厂新增“投资委员会”大功能，把收益实验室里已经通过独立前瞻、成本、基准、回撤、置信区间和多重检验的策略继续推进为“准入 → 失效检查 → 冗余检查 → 策略袖套 → 候选共识 → 漂移再平衡”。它解决的是多个局部合格策略如何组成一个整体决策，不再把每个策略各自的候选简单相加。
- 委员会只接受当前不可变收益记分卡完整性通过、当前纸面组合仍可用的策略。最近 3 个独立前瞻批次连续不跑赢基准时自动熔断；近期均值或胜率衰减但未连续失败时自动降权，历史累计好成绩不能覆盖最新失效证据。
- 策略冗余同时检查当前候选权重重叠和共同前瞻月份的成本后超额相关性；共同月份少于 4 个时明确拒绝使用相关系数，只依赖可验证的持仓重叠。委员会以等权为锚，只允许保守证据与独立贡献做窄幅倾斜，最多启用 3 个策略、单策略最高 50%。
- 单一合格策略时模型最多投入 50%；入选策略平均冗余达到 65%/80% 时最多投入 85%/70%。候选目标由“策略袖套权重 × 冻结组合内权重”合成，单一候选最高占模型 25%，所有未分配部分自动转为现金，而不是为了凑满仓位降低门槛。
- 与上一份不可变委员会指令比较策略和候选的单边换手；候选漂移低于 10% 且没有策略进出时不要求再平衡，避免小幅排名变化引发追涨杀跌。页面完整展示策略准入/淘汰理由、近期衰减、独立贡献、冗余矩阵、候选支持数、一致度、现金保留和历史指令。
- 全组合资金决策引擎升级为 `whole_portfolio_next_best_action.v2`：研究预算不再对合格策略简单等权，而是读取委员会袖套和候选上限，再继续经过已有仓位优先级、IPS、可信估值、单品/权益/行业容量和四组压力情景。委员会只输出相对优先/回避与人工研究上限，不发布伪上涨概率、不计算股数、不创建订单。
- 新增 4 个受认证操作、1 张用户隔离且拒绝 UPDATE/DELETE 的不可变指令表、`opportunity-investment-committee.v1` PostgreSQL 迁移和独立 readiness 门禁。当前 OpenAPI 共 `178` 个操作、`153` 条路径，其中机会工厂为 `20` 条路径、`24` 个操作。完整竞品映射、公式、数据模型、接口、迁移和测试记录见 [`docs/updates/2026-07-23-005-adaptive-strategy-investment-committee.md`](docs/updates/2026-07-23-005-adaptive-strategy-investment-committee.md)。
- 后端全量回归 `529 tests` 通过，前端 `npm audit` 为 `0 vulnerabilities`，Vite 生产构建完成 `1852` 个模块转换。本地真实浏览器验证了投资委员会导航、收集态、缺失值不冒充 0、首次冻结、重复冻结幂等、历史绑定、无控制台错误以及窄屏无全局横向溢出。

### 2026-07-23：投资指挥台与全组合资金决策引擎

- 首页新增“投资指挥台”，把真实持仓、当前人民币估值、有效投资政策、持仓行动报告、组合穿透风险和策略收益实验室统一为一份每日资金计划。它回答的是“已有仓位先处理什么、这期最多投入多少、投向哪些已验证候选、保留多少现金、计划后风险是否仍在边界内”，不再让用户在多个页面自行拼接结论。

### 2026-07-23：策略收益实验室与资金资格引擎

- 机会工厂新增完整的“收益实验室”，把候选扫描继续推进为“冻结前瞻批次 → 自动观察 → 成本后收益 → 同市场基准超额 → 统计门禁 → 受限人工试运行预算”。策略分数、回测收益或一次上涨都不能直接获得资金资格。
- 每个纸面组合由调度器自动追加真实行情观察；A 股、港股、美股分别使用 `510300`、`02800`、`SPY` 作为可交易基准近似，并按冻结权重计算 5/20/60 个真实交易日的精确结果。即使中途停机后补跑，也不会把更晚日期冒充目标窗口；相同行情截面使用幂等键去重。
- 收益验证统一扣除可版本化往返成本情景，检查股票/基准覆盖、成本后净收益、净超额、胜过基准比例和保守成分回撤。缺失值显示为“—”，不再被 JavaScript 错误渲染成 `0.00%`。
- 资金门禁只统计按冻结时间预先确定、起点不重叠的独立批次：5/20/60 日窗口默认至少间隔 7/28/84 个自然日。同日重复运行仍保留审计，但不能刷高样本数、胜率或置信度。
- 历史上所有真正冻结过纸面组合的策略版本都进入 Bonferroni 家族错误率校正；归档失败策略或升级版本不能抹掉试验次数，避免从大量策略中挑出随机赢家。成本、覆盖、最少 6 个成熟批次、胜基准比例、回撤和最高 5% 人工试运行比例还具有不可下调的安全底线。
- 只有主窗口全部通过才会生成非零“受限人工试运行”研究预算；金额继续受当前可核验投资政策、月度预算、可信人民币估值、允许市场和单品仓位上限约束。系统不计算股数、不创建订单、不连接券商，`execution_authorized` 始终为 `false`。
- 收益政策与记分卡使用两张用户隔离、拒绝 UPDATE/DELETE 的不可变表；今日决策研究源由 3 个升级为 4 个，并可直接跳转收益实验室。新增 6 个受认证 API、小时级 Celery 调度、`opportunity-profit-engine.v1` PostgreSQL 迁移和 readiness 门禁；当前 OpenAPI 共 `170` 个操作。
- 后端全量回归 `514 passed`、`11 subtests passed`，前端 `npm audit` 为 `0 vulnerabilities`，Vite 生产构建完成 `1850` 个模块转换。本地真实旧 SQLite 原地升级、4 研究源联动、桌面端与 `390×844` 响应式布局及浏览器控制台均通过。完整方法、公式、竞品映射和验收记录见 [`docs/updates/2026-07-23-003-opportunity-profit-engine.md`](docs/updates/2026-07-23-003-opportunity-profit-engine.md)。

| 机会工厂 | 定义不可变跨市场机会策略，运行持久扫描，查看候选漏斗、同市场五因子评分、硬门槛淘汰和约束后纸面组合；收益实验室验证精确 5/20/60 日前瞻证据；市场状态中枢按同环境前瞻样本验证策略适配并收缩风险预算；投资委员会淘汰失效/重复策略并形成候选共识模型组合 |
| 投资 Agent | 创建可恢复的单基金 Run 或 2-6 只基金 Batch，识别内地/港股/美股/全球基金，编排真实市场/持仓/新闻工具、跨基金披露持仓重合、确定性风险门禁和可选 LLM 证据合成，保存 Step、Evidence、Claim、模型调用摘要、用户决策版本和追加式审计链 |
| 系统管理 | 管理员创建和启停账户、分配角色、重置临时密码、验证不可变认证审计链，并在高可用控制中心查看组件、事故、SLO/错误预算和执行主动探测；普通用户不可见也不可调用 |

### 高可用与安全降级

- 生产流量由 Nginx 分配给 `8001/8002` 两个无状态 API 副本；单副本连接失败会被动摘除并由另一副本承接。登录会话、CSRF、幂等键和业务事实均在服务端共享存储，不依赖进程内粘性会话。
- Celery Beat 每 5 分钟记录一次固定时间桶探针，覆盖两个 API 副本、数据库、消息总线、五类 Worker、五条队列、私有对象存储和三市场专业行情；生产共 18 个组件，重复任务按稳定 ID 去重。
- 探针快照和事故事件永久追加且可校验；两次连续失败/恢复用于抑制瞬时抖动，未知状态单独保留，不会被归入正常。
- `/health/ready` 代表 API 能安全提供 PostgreSQL 中的权威事实，`/health/full` 代表全部异步能力均可用；部署监控使用后者，流量接入可使用前者。
- 用户侧能力矩阵只开放当前安全的功能。行情或队列异常时仍允许读取已保存事实，但关闭受影响的新研究、估值刷新或金额动作；权威数据库异常时整体失败关闭。
- 管理员可查看 24 小时、7 天和 30 天固定窗口 SLI、API 流量/冗余、内部目标、错误预算和事故恢复时间线。内部目标用于工程治理，不构成客户 SLA。
- 当前架构能承受一个 API 进程故障并避免依赖故障扩大为错误投资动作；真正的主机/可用区级高可用仍需跨主机副本、托管 PostgreSQL 主备、Redis 高可用、云负载均衡和异地恢复演练。

### 机会工厂

- 提供跨市场均衡、质量趋势、活跃强势验证和低波动防守四类起始模板；保存后形成完整策略版本，不会改写历史运行。
- 扫描运行在独立 `market-data` Worker，生产任务最长允许 900 秒；运行进度、成功/部分失败/失败状态、错误码和追加式事件链持久化到 PostgreSQL。
- 候选先通过真实数据可用性和风险硬门槛，再按趋势动量、估值、盈利质量、成长、风险韧性排序；总分不能掩盖任一硬门槛失败。
- 因子分位只代表“同市场、本次候选池”内的相对位置，不是全市场或行业中性排名；缺失基本面不会被静默重分配权重。
- 约束后纸面组合支持综合分×低波动、低波动或等权三种确定性权重方法，保留现金并排除高相关候选；跨市场权重暂不换算汇率。
- 纸面观察点绑定冻结基准、真实行情来源、载荷哈希、幂等键和前序事件哈希；调度器最多每日追加一次，并从真实历史中提取冻结后的精确第 5/20/60 个交易日，停机补跑不会改变目标窗口。
- 收益实验室按策略版本汇总成本后净收益、A/H/美股基准超额、胜率、95% t 区间、跨策略 Bonferroni 校正和保守成分回撤；重叠起点只保留一个预先确定的代表批次。
- 资金资格状态只有 `empty`、`collecting`、`watch`、`suspended` 和 `limited_manual_pilot`。前三种证据不足、暂停或未通过时预算固定为 0；最后一种也只生成受 IPS 与可信估值约束的人工复核金额，不授权交易。
- 市场状态中枢只聚合 14 天内、哈希通过的候选池扫描，并把当前环境与冻结纸面组合当时环境对齐；同环境样本不足时保持中性，连续失配时停用，趋势、波动和来源厚度只能降低委员会总风险上限。
- 投资委员会只接收当前不可变记分卡；三期连续失效自动停用，策略重叠/前瞻相关性降低独立贡献，单策略、重复策略和单候选集中均转为现金。候选只显示相对共识和人工研究上限，不提供伪上涨概率。
- 当前 OpenAPI 共 `182` 个操作，其中机会工厂提供 23 条策略、运行、纸面组合、收益政策、记分卡、市场状态和委员会路径（28 个操作）。

### 组合数字孪生

- 使用用户确认金额而不是模拟仓位；运行前通过 `market-data` Worker 刷新基金真实披露穿透，并用持仓哈希检查刷新期间是否发生变化。
- 提供四个可编辑的说明性情景模板；模板不是历史校准、压力发生概率或行情预测，用户可以修改五类市场冲击、行业叠加、个券总冲击、亏损预算和最小调整金额。
- 同一冻结情景同时计算当前组合与 WHAT-IF 组合；WHAT-IF 只能在现有持仓和现金之间重新分配，不允许隐含融资、卖空或新增未研究标的。
- 基金未披露或未分类权益保留为损益上下界，不填补 Beta、相关性、行业或底层持仓；页面单独显示区间宽度和未识别市场暴露。
- 在非正冲击下反推预算破线倍数；情景含正向冲击、损益可能非单调时拒绝输出单一阈值。
- 预算越线时，按最坏损失边际效率生成“减持并转现金”的最小名义金额草案与 0/25/50/75/100% 修复前沿；该草案不计税费、滑点、汇率和申赎限制，执行前必须重新核对。
- 运行表按用户隔离且不可更新、不可删除；情景、持仓、暴露、政策和结果分别计算 SHA-256，只有打开详情并完成五段复算后才显示完整性通过。

### 跨市场可信组合估值

- A 股、港股、美股股票使用真实未复权日线，基金只使用确认单位净值；USD/HKD 统一换算为 CNY，并展示实际价格、汇率来源和数据日期。
- 价格、净值和汇率写入不可变市场观察；组合估值按当前用户冻结为独立不可变快照，持仓变化、快照过期或哈希校验失败会动态关闭门禁。
- 份额不足或上游不可用时，只允许使用最近七天内的用户确认人民币金额，并明确标记为手工回退；该回退可维持组合比例复盘，但不能满足精确金额门禁。
- 风险分析要求全部持仓可估值且没有过期项；精确金额门禁还要求自动估值金额覆盖至少 95%、专业/确认行情来源覆盖至少 90%。任何门禁都不产生交易授权。
- 首页、持仓页、组合复盘、暴露、行动报告、数字孪生和 Agent 读取同一快照，避免各模块对同一组合使用不同金额口径。

### 基金研究

- 基金分类、热门榜和风险偏好机会筛选。
- 单基金净值趋势、区间收益、波动、最大回撤和恢复过程。
- 盘中估算净值上一次同值或穿越的确认净值日期区间，并保留估值与确认净值的口径差异。
- 已确认净值与盘中估值分开显示，估值不会替代确认净值。
- 同类排名、同类分位和多维替代品比较。
- 最新定期报告持仓、前后披露期变化和风格变化线索。
- 多基金相关性、重仓股/行业重合以及用户持仓穿透暴露。

### 投资 Agent 当前阶段

- 当前支持固定意图 `fund_deep_research`，读取公募基金真实数据，并可选择把用户已确认组合和投资约束纳入确定性风险门禁。
- 生产环境将 Agent Run、工具步骤、证据、结论引用、任务租约和审计事件持久化到 PostgreSQL；进程或 Worker 重启后可恢复未完成任务并复用已完成证据。本地隔离测试仍可使用 SQLite。
- 运行历史支持服务端游标分页、状态筛选、基金代码筛选和完整任务回看。
- 工具通过版本化白名单注册，当前开放 R0 公共只读工具和 R1 个人数据只读/确定性计算工具；每个工具都有实际生效的执行时限。
- 请求支持幂等键、活动队列限制、运行中取消、可选盘中估值、披露变化和同类替代品研究。
- 已完成任务可以按原配置创建新的 Run，并保留可审计的父子任务关系。
- 重跑完成后可以比较父子 Run 已持久化的指标和结论；只有双方 Evidence 与审计链均完整时才输出差异。
- 基金研究包含版本化历史条件策略：以真实净值匹配当前趋势/回撤状态，展示后续 3/6/12 个月历史分布、无条件基准和样本置信度。
- 持仓感知决策只有在投资档案、完整组合金额、风险、期限、仓位和历史条件全部通过时才计算分批研究金额；缺少关键数据时主动弃权。
- 跨境基金还必须通过市场识别、用户市场权限和汇率风险确认；QDII 估值不替代确认净值，也不直接驱动金额。
- 基金底层情报按投资市场路由：内地基金读取 A 股行业/概念，港股和美股基金使用披露行业及底层持仓行情；没有获准的海外板块源时明确标记不可用，不把 A 股板块套用到跨境基金。
- 可选大模型在所有工具完成后执行，只做证据解释、反证整理、未知项识别和复核计划；确定性代码负责精确数字、组合暴露、预算和最终允许动作。
- 模型调用记录提供商、模型、提示模板版本、输入/输出哈希、Token 使用、时延和质量门禁，但不保存 API Key 或完整原始 Prompt。
- 每条数值 Claim 绑定 Evidence，Evidence 保存来源、有效时间、质量状态和 SHA-256 摘要。
- 真实来源失败时 Run 进入 `partial` 或 `failed`，不会生成替代数据。

当前阶段没有开放式自主规划、交易工具或自动下单。LLM 仅是受约束的最终合成节点，不拥有工具权限。生产运行时已经使用 PostgreSQL、Redis/Celery 和独立 Worker，但尚未实现 PostgreSQL 行级安全、集中式身份服务、MFA 或 Temporal 级工作流编排。私人组合发送给外部模型仍默认关闭。

### 已知边界与非目标

- 技术强度是确定性历史价量规则，只描述当前状态，`decision_eligible=false`；系统不再把它映射为上涨概率。单股机器学习只保留一次时间切分的历史诊断接口，不发布最新预测，也不进入主决策界面。成本后执行回测仍需继续补充真正未见的前瞻样本、参数稳定性和分市场评测。
- 执行回测使用日线 OHLC 和用户填写的成本场景，不模拟停牌、涨跌停排队、整手限制、公司行为、汇率、融资成本或真实券商撮合，不能替代模拟盘和真实小额验证。
- 港股基本面覆盖弱于 A 股和美股。专业热门榜只有在服务端配置有效权限后才可用；富途需要常驻且已登录的 OpenD 与对应行情权限，Tushare 港股接口权限与 A 股可能不同，Alpha Vantage 实时/延时授权也需匹配订阅。未配置或授权不足时只尝试明确标记的公开降级源，云 IP 被拒绝后返回缓存或明确不可用。
- 机会工厂是确定性的研究与纸面跟踪流水线，不是开放式股票 Agent；它不提供券商连接、自动下单、盘口高频策略或收益保证。
- 组合数字孪生使用一阶金额暴露模型，不建模危机相关性变化、期权/债券全价重估、久期凸性、汇率二阶传导、流动性滑点、税费和申赎限制；预设情景也不是历史事件复现。它用于暴露脆弱点和比较动作，不是收益预测、个性化交易指令或盈利保证。
- 可信组合估值使用确认净值和日终收盘，不等于券商盘中可成交余额；参考汇率也不等于用户实际结汇价。当前未计入滑点、税费、停牌、申赎确认、公司行为、整手限制和券商现金余额。
- 内置候选池不是交易所全量，也没有历史成分股数据库；当前同市场分位可能受候选池选择和幸存者偏差影响。接入有授权的全量行情、财务和历史成分股供应商前，系统不会开放“全市场”开关。
- 本地模式默认关闭强制认证并使用 SQLite/内嵌 Worker；生产能力必须以 PostgreSQL、Redis/Celery、私有 OSS、`AUTH_REQUIRED=true` 和实际部署健康检查为准。
- 私人组合默认不发送给外部大模型；未配置模型提供商、模型 ID、Base URL 或 API Key 时，AI 合成必须保持不可用。

## 数据原则

1. 不展示伪造行情、净值、财务数据、持仓或投资结论。
2. 可以在已接入的真实来源之间切换，但必须保留实际来源、数据时间和失败原因。
3. 来源全部失败时返回 `partial`、`unavailable` 或明确错误，不用示例值兜底。
4. 用户组合分析只使用用户保存或确认过的持仓和交易。
5. 盘中估值、确认净值、日终行情和定期报告披露不得混为同一种“最新数据”。
6. 收益率、回撤、FIFO、XIRR、相关性和组合集中度由确定性代码计算，不由大模型自行编造。
7. 港股、美股、全球和 QDII 基金必须披露市场识别依据、净值时差与汇率风险；无法确认具体市场时停止金额决策。
8. 只有相关真实来源完整返回时才能自动解决旧风险；部分失败必须保留旧任务并显式记录证据缺口。
9. 任何“分数”必须说明它是规则、历史统计还是校准模型；没有前瞻验证和概率校准时不得命名为上涨概率，也不得直接进入交易动作。
10. 组合金额必须绑定当前持仓、价格/NAV、汇率、时效和完整性；旧快照、过期观察或手工金额回退不得被标记为精确可执行金额。
11. 收益验证必须取冻结后的精确第 5/20/60 个真实交易日；延迟补跑不能改变目标窗口。同窗口重叠起点不能重复计数，历史上已冻结过的策略版本不能通过归档从多重检验研究族中消失。
12. 非零资金资格必须同时通过成本、覆盖、至少 6 个独立成熟批次、成本后基准超额、胜基准比例、回撤、普通置信区间和跨策略校正区间；通过也只允许形成最高 5% 的人工复核预算，不授权下单。
13. 全组合资金计划不得跳过已有仓位行动、可信估值、当前投资政策、组合穿透或前瞻记分卡；任一绑定过期或哈希失败时新增金额必须归零。多个合格策略只能先等额分配研究袖套，再按冻结权重分配；候选行业未知时全部新增金额按同一最坏行业桶占用容量，压力计划后仍越界时继续缩减而不是忽略门禁。

## 快速开始

### 环境要求

- Python 3.10+
- Node.js 18+
- Windows 本地运行，或 Ubuntu 22.04/24.04 服务器

### Windows 一键运行

首次使用：

```text
双击 setup.bat
```

以后启动：

```text
双击 start.bat
```

浏览器会打开 [http://localhost:5173](http://localhost:5173)。后端 API 默认运行在 `http://127.0.0.1:8000`。

### 手动启动

后端终端：

```powershell
Set-Location C:\Project
.\venv\Scripts\Activate.ps1
Set-Location backend
python -m uvicorn main:app --reload --port 8000
```

前端终端：

```powershell
Set-Location C:\Project\frontend
npm install
npm run dev
```

开发环境中，Vite 会把 `/api` 请求代理到后端 `8000` 端口。FastAPI 接口文档位于 [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)。

本地开发默认保持 `AUTH_REQUIRED=false`，以兼容已有的一键启动流程；这只适用于本机开发。生产 systemd 模板固定启用认证，并且在审计 Pepper 或初始管理员缺失时返回 `503`，不会匿名放行。初始化生产管理员的完整步骤见 [DEPLOY.md](DEPLOY.md)。
